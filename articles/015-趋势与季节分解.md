# 趋势与季节——拆解一个时序

> 一个电商卖家的月销售额，能拆成"增长势头""季节性节律""随机波动"三部分。理解了这三者如何交织，你就理解了时间序列的核心直觉。

---

## 📦 一个电商卖家的月销售额

假设你经营一家小电商——卖手冲咖啡器具。

你的月销售额数据从 2019 年 1 月记录到 2022 年 12 月，一共 48 个月。画出来是这样的：

![电商月销售额原始序列](images/015-raw-series.png)
*图 1：某电商平台月销售额（2019-2022）。有上涨趋势、有季节波动——2月跌（春节）、6月跳（618促销）、11-12月冲顶（双11+双12）。*

用肉眼看，这个序列里有**三种信号**在同时作用：

1. **趋势（Trend）**——店面越做越好，销售额总体在往上走
2. **季节（Seasonal）**——每年固定在春节跌、6月和年末涨
3. **噪声（Noise）**——有些月份涨跌莫名其妙，比如某个月的物流出问题了

**时间序列分解就是把一个时序拆成这三部分。**

从直觉上讲，这就像你听一首交响乐——你可以单独去听大提琴（趋势），也可以注意小提琴的旋律（季节），还有定音鼓的偶尔爆发（噪声）。

---

## 🧩 加法模型 vs 乘法模型

把时序拆开，有两种基本思路。

### 加法模型：趋势 + 季节 + 噪声

```
Y(t) = Trend(t) + Seasonal(t) + Residual(t)
```

每部分用"绝对量"来表达。比如：

- 趋势是这个月的**基准销售额**，100 万
- 季节是每年 12 月**额外加的** 20 万
- 噪声是随机**±5 万**

加法模型意味着：季节的振幅不随趋势的增长而变化。即使你的销售额从 100 万涨到了 300 万，12 月的"额外"还是 20 万。

### 乘法模型：趋势 × 季节 × 噪声

```
Y(t) = Trend(t) × Seasonal(t) × Residual(t)
```

每部分用"比例"来表达。比如：

- 趋势是这个月的**基准销售额**，100 万
- 季节是每年 12 月**乘以 1.2 倍**
- 噪声是随机**乘以 0.98~1.02**

乘法模型意味着：季节的振幅随趋势的增长而等比放大。当你的销售额从 100 万涨到 300 万，12 月的增量就从 20 万变成了 60 万。

### 什么时候用哪个？

这是一个重要的选择。经验法则很简单：

> **如果季节波动幅度随着趋势增长而变大 → 用乘法模型**
> **如果季节波动幅度保持稳定 → 用加法模型**

对于电商数据：当你的店还小时，双 11 多卖 20 万；当你的店做大了，双 11 可能多卖 200 万。这种情况下**乘法模型**更自然。

但我们先从加法模型开始看——它更直观，易于理解。

## 🔬 statsmodels 怎么做到的？——移动平均

你可能会好奇：计算机是怎么从一串数字里"拆"出趋势和季节的？

### 第一步：提取趋势——移动平均

思想极其直观：**把过去 12 个月的销售额取个平均值，抹掉季节和噪声。**

```python
import pandas as pd
# 对月度数据做 12 步移动平均
trend_estimate = df['sales'].rolling(window=12, center=True).mean()
```

这叫做**中心化移动平均（Centered Moving Average, CMA）**。为什么是 12？因为数据周期为 12 个月。移动平均把整年的波动均匀了——12 个月的峰值和低谷相互抵消，剩下的就是基础趋势。

如果你怀疑你的数据是不是足够"干净"来提取趋势——**不需要**。移动平均对季节和噪声都有很好的抗性，数据够长了就能用。

> **关于边界丢失**：用中心化移动平均时，两端会丢失大约 `window/2` 个数据点。如果你的序列很短，可以考虑 `extrapolate_trend=True` 参数或者用别的平滑方法。

### 第二步：去趋势——得到"季节 + 噪声"

从原始序列中减去（在乘法模型中则是除以）趋势：

```python
detrended = df['sales'] - trend_estimate
```

现在趋势被移除了，剩下的就是季节和噪声。

### 第三步：提取季节——按月份平均

把 4 年中每年 1 月的"去趋势值"取平均，2 月的取平均……直到 12 月的取平均。

```python
# 伪代码：按月份取平均
seasonal = detrended.groupby(by=month).mean()
```

这样拆出来的就是**固定的季节模式**——每年 1 月比趋势高多少，2 月比趋势低多少。

### 第四步：残差就是剩下的

从原始序列中减去趋势和季节，剩下的就是残差（纯随机噪声）：

```python
residual = df['sales'] - trend_estimate - seasonal
```

整个过程就是这四步——非常简单、非常直观。

## 🐍 Python 实战：现在动手拆一个时序

我们用模拟的电商数据来演示。

### 模拟数据

```python
import numpy as np
import pandas as pd
import matplotlib.pyplot as plt
from statsmodels.tsa.seasonal import seasonal_decompose

# 模拟48个月电商销售额
np.random.seed(42)
months = pd.date_range(start='2019-01-01', periods=48, freq='M')
t = np.arange(48)

# 趋势：线性增长 + 轻微加速
trend = 100 + 3.5 * t + 0.04 * t**2

# 季节：12个月周期 + 促销脉冲
seasonal_pattern = (20 * np.sin(2 * np.pi * t / 12 + 1.2) 
                    + 10 * np.sin(2 * np.pi * t / 6 + 0.5))
# 增强年末效应
for i in range(48):
    m = i % 12
    if m == 10:  seasonal_pattern[i] += 25   # 11月双11
    elif m == 11: seasonal_pattern[i] += 20  # 12月双12
    elif m == 5:  seasonal_pattern[i] += 15  # 6月促销
    elif m == 1:  seasonal_pattern[i] -= 15  # 春节低点

# 噪声
noise = np.random.normal(0, 8, 48)

# 合成加法序列
sales = trend + seasonal_pattern + noise
```

### 一行代码分解

```python
from statsmodels.tsa.seasonal import seasonal_decompose

decomp = seasonal_decompose(sales, model='additive', period=12)

# 四个部分就出来了
decomp.observed   # 原始序列
decomp.trend      # 趋势分量
decomp.seasonal   # 季节分量
decomp.resid      # 残差分量
```

输出结果一目了然——**四个时间序列并排**：

![季节分解图](images/015-decomposition.png)
*图 2：加法模型分解结果——Observed → Trend + Seasonal + Residual。注意趋势图里那条灰色虚线是"真实趋势"（我们模拟时已知的），红色的线是分解出来的趋势估计。两者高度吻合！*

### 解读分解图

当你看到分解图，你应该问自己三个问题：

1. **趋势是一条平滑的曲线吗？** 如果不是，说明 period 选得不对或数据不够长。
2. **季节分量每年的模式一致吗？** 如果某些年份的季节模式变了，可能需要分段分解。
3. **残差看起来像白噪声吗？** 如果残差还有明显模式，说明模型欠拟合——可能有更长的周期没捕捉到。

在我们的分解结果中：
- 趋势曲线平滑地向上，反映了店铺的持续增长
- 季节分量每年波形几乎一致，12 月最高、2 月最低
- 残差围绕 0 随机波动，没有明显模式

**这就是一个"好分解"的标志。**

## ⚖️ 加法 vs 乘法：用同一个数据对比

前面我们说过，**当季节的振幅随趋势增长时，乘法模型更合适**。

为了验证这一点，我模拟了一个"季节振幅随趋势增长"的时序：

```python
# 乘法模型数据：季节和噪声是比例因子
sales_mul = trend * (1 + seasonal_pattern / np.mean(trend) * 0.6 + noise / np.mean(trend) * 0.15)
```

然后在同一页上并排对比加法分解和乘法分解：

![加法 vs 乘法](images/015-additive-vs-multiplicative.png)
*图 3：加法模型（上排）和乘法模型（下排）对同一个"季节振幅随趋势增长"的时序的分解对比。*

注意几个关键差异：

| 特征 | 加法模型 | 乘法模型 |
|------|---------|---------|
| 季节分量 | 围绕 0 波动（±固定值） | 围绕 1 波动（比例因子） |
| 趋势形态 | 线性平滑 | 线性平滑（双方法对趋势估计接近） |
| 适用场景 | 季节幅度稳定 | 季节幅度随趋势放大 |

在乘法分解中：
- 季节分量值围绕 1 波动——12 月为 1.2（+20%），2 月为 0.85（-15%）
- 这意味着 12 月的增量随着销售额基数的增长而等比放大
- 这更符合真实商业场景

**实战建议**：如果你拿不准用哪个，可以两种都跑一遍，看哪个的残差更接近白噪声（残差方差更小、没有明显模式）。

## 🧠 背后更深的一层：为什么要拆？

时间序列分解看起来只是把一条曲线拆成三条曲线。但它的思想比你想象的更深刻。

### 拆解 = 理解因果

趋势是谁在驱动？可能是用户增长、品牌溢价、市场扩张。
季节是谁在驱动？可能是天气、节假日、促销日历。
噪声是谁在驱动？可能是物流罢工、竞争对手大促、黑天鹅事件。

**每一种分量背后都有一个对应的商业或经济原因。** 如果你不去拆，你永远只能看到一个模糊的"涨了"或"跌了"。

### 拆解 = 更好的预测

知道趋势是上升的、季节在 12 月有高峰，你可以：

1. 用趋势的线性外推估计下个月的基础销售额
2. 加上对应月份的季节效应
3. 得到比"直接预测原序列"更稳定、更可解释的预测

### 拆解 = 异常检测

残差就是"已知规律之外的东西"。如果你发现某个月的残差特别大（比如 3 个标准差以上），大概率出了什么事——可能是数据错误、运营事故、或者一个需要关注的外部冲击。

## 🔗 延伸：量化交易中的因子时序拆解

时间序列分解不仅在电商分析中有用。**它在量化投资中是一个基础工具。**

### 因子的趋势与季节

假设你构建了一个"A股低波因子"，你每个月计算因子收益，得到一条时间序列。

你同样可以把它拆成：

- **趋势**：这个因子的长期有效性是否在衰减？（低波因子在 A 股的有效性最近几年确实在下滑）
- **季节**：因子是否有特定月份表现更好？（比如"1月效应"、"春节效应"）
- **噪声**：因子收益的随机波动

### 因子择时

这是量化领域的一个重要方向——**因子择时（Factor Timing）**。

核心思想：如果因子收益的"趋势"和"季节"成分可以用基本面或宏观变量来解释，那就可以在这些成分走弱时减仓、走强时加仓。

举个例子：
- 低波因子的趋势在牛市中可能走弱（大家都追高，不在乎波动）
- 低波因子的趋势在熊市中可能走强（资金避险）
- 季节性地，季末机构调仓可能会放大因子的波动

如果不对因子收益做分解，你只能看到一条上下波动的曲线，不知道什么变化是结构性的、什么是周期性的、什么是纯粹的噪声。

**更深入地理解因子的成分，就是在更深入地理解市场的运行逻辑。**

```python
# 量化因子收益的时间序列分解（伪代码）
factor_returns = get_factor_returns('low_beta', start='2010', end='2024')
decomp = seasonal_decompose(factor_returns, model='additive', period=12)

# 趋势是否在衰减？
if decomp.trend.iloc[-12:].mean() < decomp.trend.iloc[-24:-12].mean():
    print("⚠️ 低波因子的长期有效性在下滑")
    
# 季节效应是否显著？
seasonal_amplitude = decomp.seasonal.max() - decomp.seasonal.min()
if seasonal_amplitude > factor_returns.std() * 0.5:
    print(f"📅 因子存在显著季节效应（振幅={seasonal_amplitude:.2%})")
```

## ✅ 小结

这一篇我们学到了：

- **时间序列分解**把一个序列拆成 Trend + Seasonal + Residual
- **加法模型**适用于季节幅度稳定的场景，**乘法模型**适用于季节幅度随趋势增长而放大的场景
- **statsmodels 的 seasonal_decompose** 一行代码完成分解——背后是移动平均 + 按周期平均的简单思想
- **分解的价值**：理解因果、辅助预测、异常检测
- **量化延伸**：因子收益也可以做趋势-季节-残差三分解，是因子择时的重要工具
