# Logit 回归——是"是"还是"否"

> 银行审批贷款，究竟怎么判断一个人"会"还是"不会"违约？这个问题从本质上就和之前我们研究的所有问题不一样——因为答案只有两个：是，或者否。

---

## 🏦 一个非黑即白的问题

之前我们做了十几篇回归分析，**因变量全都是连续数值**：房价（万元）、消费（千元）、股价收益率（百分比）……

现在情况变了。**因变量只有两个取值**——是/否、违约/不违约、点击/不点击、患病/不患病。

比如银行想知道：**某个客户会不会违约？**

- 因变量 $Y = 1$（违约）/ $Y = 0$（不违约）
- 自变量 $X$：收入、负债率、信用分、年龄……

如果你拿起 OLS 就往上套，会有什么后果？

**让我们来亲手试试——这就是著名的"线性概率模型"（Linear Probability Model, LPM）。**

---

## ❌ 线性概率模型——看似合理，实则灾难

### 思路

既然 $Y$ 是 0 或 1，那 $E[Y|X]$ 就是"给定 X 时 Y=1 的概率"。

为什么？回忆离散随机变量的期望：

$$
E[Y|X] = 1 \times P(Y=1|X) + 0 \times P(Y=0|X) = P(Y=1|X)
$$

所以用 OLS 去拟合 $Y$，本质上就是在拟合概率 $P(Y=1|X)$。听起来合理？

### 模拟数据

我们模拟银行客户的违约数据：用负债率（Debt-to-Income Ratio）来预测是否违约。

```python
import numpy as np
import pandas as pd
import statsmodels.api as sm
import matplotlib.pyplot as plt

np.random.seed(42)
n = 500

# 核心驱动变量：负债率（DTI）
dti = np.random.uniform(0.1, 0.8, n)

# 真实违约概率：负债率越高，违约概率越大（非线性）
logit_p = -3 + 5 * dti
prob = 1 / (1 + np.exp(-logit_p))

# 生成二值结果
default = np.random.binomial(1, prob)

df = pd.DataFrame({'dti': dti, 'default': default, 'prob': prob})
```

### 用 OLS 拟合

```python
X = sm.add_constant(df['dti'])
lpm = sm.OLS(df['default'], X).fit()
df['lpm_fitted'] = lpm.fittedvalues

print(lpm.summary().tables[1])
```

输出（简化）：

```
────────────────────────────────────────────
             coef    std err     t     P>|t|
────────────────────────────────────────────
const      -0.181      0.058  -3.13   0.002
dti         0.857      0.116   7.40   0.000
────────────────────────────────────────────
```

截距 -0.18，dti 系数 0.86。这意味着：

- 负债率 = 0.2 时，预测违约概率 ≈ -0.18 + 0.86×0.2 = **-0.008** → 负数！
- 负债率 = 0.8 时，预测违约概率 ≈ -0.18 + 0.86×0.8 = **0.508** → 还行
- 极端情况下，预测值可以超过 1

**你的"概率"变成了负数——这显然不对。**

![LPM vs Logit 对比](https://cdn.jsdelivr.net/gh/qumin86753649iiu/zhihu-econometric/images/027-lpm-vs-logit.png)
*图 1：红色虚线是 LPM 的拟合值，蓝色 S 曲线是 Logit。可以看到 LPM 在两端越过了 [0,1] 的合理区间——预测概率可能是负数，也可能大于 1。Logit 则天然被限制在 0 和 1 之间。*

## ✅ Logit 回归——S 曲线把概率"夹"在 [0,1] 之间

### 核心思想

与其用一条直线去硬拟合二元变量，不如找一个**值域天然在 (0,1) 之间的函数**。

逻辑函数（Logistic Function）：

$$
P(Y=1|X) = \frac{1}{1 + e^{-(β₀ + β₁X₁ + \cdots + βₖXₖ)}}
$$

这就是 **S 曲线（Sigmoid Curve）**：

- 当 $β₀ + β₁X₁ + \cdots → -∞$，$P → 0$
- 当 $β₀ + β₁X₁ + \cdots → +∞$，$P → 1$
- **永远不越界**

![Logit S 曲线](https://cdn.jsdelivr.net/gh/qumin86753649iiu/zhihu-econometric/images/027-logit-curve.png)
*图 2：Sigmoid 函数将线性组合映射到 (0,1)。决策边界通常在 P=0.5 处——如果你预测的概率大于 50%，就说"会违约"。*

### 另一种等价的写法——对数几率（Log-Odds）

把上面的公式做一个小小的变换，就得到了 Logit 的本质：

$$
\ln\left(\frac{P}{1-P}\right) = β₀ + β₁X₁ + \cdots + βₖXₖ
$$

左边是**对数几率（Log-Odds）**——几率（Odds）是 $P/(1-P)$，表示"违约概率 vs 不违约概率的比值"。

**好处**：右边回到了我们熟悉的线性形式，系数可以直接解释为"X 每增加一个单位，对数几率变化 β"。

---

## 🐍 Python 实战：用 statsmodels 拟合 Logit

### 第一步：拟合 Logit 模型

```python
import statsmodels.formula.api as smf

# 方法一：用 formula API（推荐，直观）
logit_mod = smf.logit('default ~ dti', data=df)
logit_res = logit_mod.fit()

print(logit_res.summary())
```

输出（简化）：

```
──────────────────────────────────────────────
                 coef    std err     z    P>|z|
──────────────────────────────────────────────
Intercept    -2.8517      0.328  -8.69   0.000
dti           4.7861      0.709   6.75   0.000
──────────────────────────────────────────────
```

**解读方法**：
- 负债率每提高 1 个单位，违约的对数几率增加 4.79
- 对数几率不好理解——所以我们接下来换成更好懂的指标

### 第二步：几率比（Odds Ratio）

几率比 = $e^{β}$，表示"X 每增加一单位，违约的几率（odds）变为原来的几倍"。

```python
odds_ratio = np.exp(logit_res.params)
print(odds_ratio)
```

输出：

```
Intercept    0.058
dti        119.832
```

负债率每提高 1 个单位，**违约的几率变为原来的约 120 倍**。

> ⚠️ 注意：这里的"1 个单位"如果对负债率来说太大（负债率范围是 0~1），你可以标准化或报告"每增加 0.1 个单位"的效果：
>
> ```python
> # 每增加 0.1 个单位的几率比
> np.exp(logit_res.params['dti'] * 0.1)  # ≈ 1.61
> ```
>
> 负债率每提高 0.1，违约几率增加约 61%，这样更直观。

**为什么几率比比原始系数好？**
- 原始系数在"对数几率"的尺度上，人类不直观
- 几率比在"倍数"尺度上——"变为了 X 倍"人人都懂

### 第三步：边际效应（Marginal Effects）

几率比虽然好理解，但它回答的是"几率的倍数变化"——而不是"概率本身的变化"。

很多时候你想知道的是：**负债率提高 0.1，违约概率究竟增加几个百分点？**

这就需要计算边际效应。

在 Logit 中，边际效应不是常数——它依赖于当前 X 的取值。最常用的方法是**边际效应在均值处（Marginal Effect at the Mean, MEM）**：

```python
from statsmodels.discrete.discrete_model import Logit
from statsmodels.tools import add_constant as add_const

# 计算边际效应（在均值处）
X_mean = df[['dti']].mean().values.reshape(1, -1)
X_mean_with_const = add_const(pd.DataFrame(X_mean, columns=['dti']))

# 方法一：statsmodels 内置
margeff = logit_res.get_margeff(at='mean')
print(margeff.summary())
```

输出（简化）：

```
────────────────────────────────
           dy/dx   Std.Err.   z
────────────────────────────────
dti       0.699     0.069    10.12
────────────────────────────────
```

**负债率每提高 1 个单位，违约概率增加约 70 个百分点。**（如果按 0.1 单位算，约 7 个百分点。）

**三种指标对比：**

| 指标 | 含义 | 数值（dti 系数） |
|------|------|-----------------|
| 原始系数 (Log-Odds) | 负债率每+1，对数几率增加 | 4.79 |
| 几率比 (Odds Ratio) | 负债率每+1，违约几率变为 | 119.83 倍 |
| 边际效应 (MEM) | 负债率每+1，违约概率增加 | 69.9 个百分点 |

**最推荐什么？** 论文和报告中最好同时报告几率比（解释强度）和边际效应（解释实际影响量级）。

![边际效应可视化](https://cdn.jsdelivr.net/gh/qumin86753649iiu/zhihu-econometric/images/027-marginal-effects.png)
*图 3：蓝色 S 曲线是 Logit 的预测概率。红色虚线表示在均值负债率处的切线斜率——这就是边际效应。注意斜率在不同位置不同：在中间最陡（对概率影响最大），在两端接近 0。*

## 🤖 延伸：AI 分类器的概率校准

你可能注意到一个有趣的事：**Logit 回归本身就是一个分类器。**

实际上，逻辑回归现在是机器学习中最基础的分类模型之一。但今天几乎所有的 AI 分类器（随机森林、XGBoost、神经网络）**输出的"置信度"往往有偏**。

什么意思？

以 XGBoost 为例，它输出的"概率"常常是：
- 预测 0.9 的样本中，真实占比可能只有 0.7（过度自信）
- 预测 0.3 的样本中，真实占比可能达到 0.5（不够自信）

**如何修正？** 用 Logit 回归做概率校准（Platt Scaling）。

### 方法

1. 先拿 XGBoost 对验证集做预测，得到原始得分（raw scores）
2. 然后用 **Logit 回归** 把这些得分重新校准：

```python
# 假设 xgb_probs 是 XGBoost 的原始预测
from sklearn.linear_model import LogisticRegression

calibrator = LogisticRegression()
calibrator.fit(xgb_scores.reshape(-1, 1), y_val)

# 校准后的概率
calibrated_probs = calibrator.predict_proba(xgb_scores.reshape(-1, 1))[:, 1]
```

本质就是用一个 Logit 模型去拟合"真实概率 vs 原始得分"的关系——把 S 曲线重新套在了分类器的输出上。

**这是什么？这就是计量经济学中的 Logit 回归，以"概率校准器"的身份进入了 AI 的 pipeline。**

### 可靠性图（Reliability Diagram）

校准好坏通常用**可靠性图**来检查：

- 横轴：预测概率（分桶）
- 纵轴：实际正例比例
- 完美校准的对角线：y = x

```
预测概率桶     实际正例比例   偏差
─────────────────────────
0.0 - 0.2        0.18      0.02
0.2 - 0.4        0.35      0.05
0.4 - 0.6        0.55      0.05
0.6 - 0.8        0.72      0.08
0.8 - 1.0        0.85      0.15 ← 过度自信
```

校准前偏差大 → 用 Logit 做 Platt Scaling → 偏差变小。

**其实 AI 工程师每天都在用计量工具，只不过他们不一定意识到这一点。**

## ✅ 小结

这一篇我们学到：

- **线性概率模型（LPM）**：用 OLS 拟合二元变量，简单粗暴，但预测值会超出 [0,1]，本质上不合理
- **Logit 回归**：用 S 曲线把概率限制在 [0,1] 之间，是最常用的二元分类模型
- **三种解读方式**：原始系数（对数几率）→ 几率比（倍数）→ 边际效应（百分点），各有适用场景
- **概率校准**：AI 分类器的输出常常有偏，用 Logit（Platt Scaling）可以重新校准

**核心心法**：Logit 回归把"是/否"问题转化为"比率的对数"问题——用了一个漂亮的非线性变换，把离散的世界和线性模型桥接了起来。
