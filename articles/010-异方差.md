# 异方差——误差的方差不再是常数

> 有人花钱精打细算，有人花钱随心所欲——收入越高的人消费波动越大。你的模型能看到这个区别吗？

---

## 💳 月薪三千和月薪三万的人怎么花钱？

假设你现在在研究一个问题：**收入和消费之间的关系。**

你去街上随机找了 200 个人，问他们的月收入和月消费。简单线性回归，画出来大概是这样：

![模拟数据：收入 vs 消费](https://cdn.jsdelivr.net/gh/qumin86753649iiu/zhihu-econometric/images/010-income-consumption-scatter.png)
*图 1：横轴是月收入，纵轴是月消费。现在你用肉眼先看一下——有什么不对劲吗？*

大约一半的点在回归线附近，另一半......

**月薪三千的人的点密密麻麻挤在回归线附近——大家消费习惯很相似，省着花。月薪三万的人的点则四处散落——有人买包买车，有人存钱理财，方差大得多。**

这就是异方差——**误差的方差不再是一个常数，而是随着某个变量的变化而变化。**

---

## 🔍 什么是异方差？为什么它在真实世界中无处不在？

### 同方差 vs 异方差

回顾第 4 篇我们讲过的 OLS 的假设：误差 ε 的方差应该是常数（同方差，Homoscedasticity）。

```
同方差：Var(ε|X) = σ²    ← 方差是常数
异方差：Var(ε|X) = σ²(X)  ← 方差随 X 变化
```

在消费这个例子里，误差就是"实际消费 - 预测消费"。

- 对低收入人群：这个差距很小且稳定（大家消费行为相似）
- 对高收入人群：这个差距大且不稳定（大家消费行为分化）

### 异方差在什么时候容易出现？

我总结了三类常见场景：

1. **横截面数据中，量级差异大的变量**：收入、财富、企业规模、人口——量级大意味着波动大
2. **分组特征**：不同群体内部的变异程度天然不同
3. **时间序列中的波动率聚集**：这个后面第 18 篇 GARCH 会深讲

### 为什么你该在意它？

异方差不会让 **系数估计值本身** 有偏——点估计还是对的。

**但它会让标准误出错。** 标准误错了，t 值就错了，p 值就错了，你的显著性判断就不可靠了。

什么意思？就是你自信满满地说"这个变量显著！"，但其实它是假的。

> 类比做菜：配方（系数）是对的，但温度计（标准误）坏了——你不知道菜到底熟了没有。

---

## 📐 数学：只是标准误的问题

在同方差下，OLS 系数的方差公式很漂亮：

```
Var(β̂) = σ²(X'X)⁻¹
```

σ² 是个常数，一切都很优美。

在异方差下，公式变成了更一般的形式（White 1980）：

```
Var(β̂) = (X'X)⁻¹ X' diag(σ²ᵢ) X (X'X)⁻¹
```

我们不做推导。你只需要知道：

- 同方差是"特殊情况"——它很美但很少见
- 异方差是"一般情况"——它更丑但更真实
- **OLS 的系数没变，只是标准误要换成"稳健标准误"（Robust Standard Errors）**

## 🐍 Python 实战：一步步诊断异方差

### 第一步：跑一个 OLS，然后画残差图

```python
import pandas as pd
import numpy as np
import statsmodels.api as sm
import matplotlib.pyplot as plt

# 模拟数据（含异方差）
np.random.seed(42)
n = 300
income = np.random.uniform(3, 50, n)  # 月收入（千元）
# 异方差：消费的波动随收入增加
consumption = (
    3 + 0.6 * income
    + np.random.normal(0, 1, n) * (1 + 0.3 * income)
)

df = pd.DataFrame({'income': income, 'consumption': consumption})

# 跑 OLS
X = sm.add_constant(df['income'])
m = sm.OLS(df['consumption'], X).fit()
df['resid'] = m.resid
df['fitted'] = m.fittedvalues

# 残差图
fig, axes = plt.subplots(1, 2, figsize=(12, 5))

# 左：残差 vs 拟合值
axes[0].scatter(df['fitted'], df['resid'], alpha=0.4)
axes[0].axhline(y=0, color='gray', linestyle='--')
axes[0].set_xlabel('拟合值（预测消费）')
axes[0].set_ylabel('残差')
axes[0].set_title('残差 vs 拟合值')

# 右：残差 vs 收入（异方差的来源变量）
axes[1].scatter(df['income'], df['resid'], alpha=0.4)
axes[1].axhline(y=0, color='gray', linestyle='--')
axes[1].set_xlabel('月收入（千元）')
axes[1].set_ylabel('残差')
axes[1].set_title('残差 vs 收入')
```

![残差诊断图](https://cdn.jsdelivr.net/gh/qumin86753649iiu/zhihu-econometric/images/010-residual-diagnostics.png)
*图 2：左图残差 vs 拟合值，右图残差 vs 收入。两张图都显示一个清晰的"喇叭形"——拟合值（收入）越大，残差的散开范围越大。这就是异方差的典型标志。*

### 第二步：Breusch-Pagan 检验——用统计量确认直觉

肉眼看到"喇叭形"还不够——我们要一个客观的检验。

**Breusch-Pagan 检验的思想**：把残差的平方作为因变量，用原来的自变量再做一次回归。如果自变量能显著地"解释"残差平方的变化（即方差的变化），就说明存在异方差。

```python
from statsmodels.stats.diagnostic import het_breuschpagan

bp_test = het_breuschpagan(m.resid, X)
print(f"LM 统计量: {bp_test[0]:.2f}")
print(f"LM p值:    {bp_test[1]:.4f}")
print(f"F 统计量:  {bp_test[2]:.2f}")
print(f"F p值:     {bp_test[3]:.4f}")
```

输出：

```
LM 统计量: 59.53
LM p值:    0.0000
F 统计量:  73.78
F p值:     0.0000
```

**p < 0.001——强烈拒绝"同方差"的原假设。** 也就是说：异方差确实存在。

稍微多解释一句：
- **原假设 H₀**：同方差（Var(ε|X) = 常数）
- **备择假设 H₁**：异方差
- p < 0.05 → 拒绝 H₀ → 存在异方差

### 第三步：用稳健标准误修正

找到了问题，怎么修？

你不需要改模型，只需要让标准误"稳健"。

```python
# 使用 HC3（小样本修正版，推荐）
m_robust = m.get_robustcov_results(cov_type='HC3')
print(m_robust.summary())
```

对比一下修正前后的标准误：

```
─────────────────────────────────────
变量       系数    普通SE  稳健SE  变化%
─────────────────────────────────────
const     2.20    1.244    0.857   -31.1%
income    0.638   0.042    0.046    +8.7%
─────────────────────────────────────
```

注意看两个标准误的变化方向不同：
- **截距的标准误变小了**（-31.1%）——对截距的估计反而更精确了
- **收入的标准误变大了**（+8.7%）——对收入效应的估计更保守了

这说明异方差对不同系数的影响方向不同，不能一概而论。**稳健标准误就是在你不知道异方差如何影响每个系数时，给你一个统一的修正方案。**

![标准误对比](https://cdn.jsdelivr.net/gh/qumin86753649iiu/zhihu-econometric/images/010-standard-errors-comparison.png)
*图 3：横轴是变量，纵轴是标准误。浅色是普通标准误，深色是稳健标准误（HC3）。两个系数的标准误都发生了变化——收入的标准误变大，截距的变小。稳健标准误给你的不是一个"更大"的答案，而是一个"更正确"的答案。*

### 第四步：完整的报告模板

当你下次做回归分析时，至少这样输出：

```python
def regression_report(m_ols, cov_type='HC3'):
    """输出带稳健标准误的回归结果"""
    m_robust = m_ols.get_robustcov_results(cov_type=cov_type)
    
    # BP 检验
    bp = het_breuschpagan(m_ols.resid, m_ols.model.exog)
    has_hete = "⚠️ 存在异方差" if bp[1] < 0.05 else "✅ 同方差"
    
    print(f"""
    ╔══════════════════════════════════════╗
    ║  回归报告 + 异方差诊断                ║
    ╠══════════════════════════════════════╣
    ║  模型: OLS  (稳健标准误: {cov_type})      ║
    ║  BP检验: LM={bp[0]:.2f}, p={bp[1]:.4f}   ║
    ║  诊断:   {has_hete}                  ║
    ║  R² = {m_ols.rsquared:.3f}                      ║
    ╠══════════════════════════════════════╣
    """)
    print(m_robust.summary())
```

**记住：写论文、做报告、跑策略，都用稳健标准误。** 它不影响系数，只让标准误更诚实。在真实世界中，稳健标准误应该是默认选项，而不是"额外加上的奢侈品"。

## 🔗 延伸：量化中的波动率异方差

异方差不仅仅存在于消费数据中。**金融数据几乎是天然异方差的。**

回想一下股票日收益率：

- 平静期：日波动 ±1%
- 波动期：日波动 ±5%

这就是**波动率聚集**——大的变化往往跟着大的变化，小的变化跟着小的变化。GARCH 模型（第 18 篇）就是专门处理这类异方差的时间序列模型。

在量化策略中，**忽略异方差会让你的回测结果看起来太好**：

- 策略夏普比率被高估
- t 值被高估
- 实际上不显著的因子被误认为有效

**所以：做因子回归，一定要用稳健标准误。**

这在 statsmodels 中只需加一个参数：

```python
sm.OLS(y, X).fit(cov_type='HAC', cov_kwds={'maxlags': 5})
```

- **HC**: Heteroscedasticity-Consistent（异方差稳健）
- **HAC**: Heteroscedasticity and Autocorrelation Consistent（异方差+自相关稳健）——时间序列中用

## ✅ 小结

这一篇我们学到：

- **异方差** = 误差的方差不是常数——真实世界中的常态
- **危害**：标准误被低估，显著性被高估，系数本身没问题
- **诊断**：画残差图（看有没有"喇叭形"）+ Breusch-Pagan 检验
- **修正**：用稳健标准误（HC/HAC）——一行代码的事
- **量化延伸**：金融数据天然异方差，因子回归必须用稳健标准误
