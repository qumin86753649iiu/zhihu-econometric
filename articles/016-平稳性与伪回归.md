# 平稳性——伪回归的源头

> 中国出生率在不断下降，而某只美国科技股在过去十年涨了十倍。你把两者做个回归——显著！中国出生率每下降1‰，这只股票就涨5%。这是真的吗？

---

## 🎭 天大的巧合：两个完全不相关的时间序列

假设你现在干一件荒唐的事：把**中国每年的出生率**和**美国某科技公司（比如苹果）的股价**放在一起做个回归。

你会得到什么？

```
apple_price = β₀ + β₁ × birth_rate + ε
```

如果你真的做了这个回归——不夸张地说——你很可能得到一颗**显著的星星**：p < 0.05，R² 还挺高。

但这就好比说"地球自转速度的变化决定了比特币的价格"——听起来像阴谋论，但回归结果会"证实"它。

这到底是怎么回事？

**答案是：两个序列都是非平稳的（Non-stationary）。它们在各自的方向上随心所欲地游荡，但回归算法会误以为它们之间存在关联。**

这就是计量经济学历史上最臭名昭著的陷阱之一——**伪回归（Spurious Regression）**。

### 伪回归的经典故事

1974 年，计量经济学家 Clive Granger 和 Paul Newbold 做了一个实验：

他们生成了**两个完全独立的随机游走**：

```
Xₜ = Xₜ₋₁ + ε₁ₜ
Yₜ = Yₜ₋₁ + ε₂ₜ
```

其中 ε₁ₜ 和 ε₂ₜ 是独立的白噪声——X 和 Y 理论上毫无关系。

然后他们把 X 对 Y 做回归。结果呢？

**在 5% 的显著性水平下，竟然有大约 75% 的概率得到一个"显著"的结果！**

换句话说：两个毫无关系的随机游走，有 3/4 的概率会被误认为有显著的相关关系。

> 💡 **关键洞察**：非平稳序列的"伪相关"不是因为数据本身有问题，而是因为回归的统计推断（t 检验、F 检验）在非平稳数据下全部失效了。

## 🔢 模拟：亲眼看看伪回归

让我们用 Python 生成两个独立的随机游走，然后看看回归结果有多"好"。

```python
import numpy as np
import pandas as pd
import statsmodels.api as sm
import matplotlib.pyplot as plt

np.random.seed(42)
n = 200

# 两个独立的随机游走
e1 = np.random.randn(n)
e2 = np.random.randn(n)
X = np.cumsum(e1)        # 随机游走 1
Y = np.cumsum(e2)        # 随机游走 2

# 做回归：Y = β₀ + β₁X + ε
X_with_const = sm.add_constant(X)
m = sm.OLS(Y, X_with_const).fit()

print(m.summary())
```

你猜结果会怎样？

```

Model:                            OLS   Adj. R-squared:                  0.241
Method:                 Least Squares   F-statistic:                     64.25
Date:                ...                Prob (F-statistic):           1.23e-13
─────────────────────────────────────────────────────────────────
       coef     std err          t      P>|t|
─────────────────────────────────────────────────────────────────
const -0.3449      0.340     -1.014      0.312
X1     0.3805      0.047      8.016      0.000   ← *** 高度显著！
─────────────────────────────────────────────────────────────────
```

看！p ≈ 0.000，t 统计量超过 8，R² = 0.245——一切都看起来"完美"。

**但这两个序列是我们故意生成的、完全独立的白噪声累积。它们之间根本没有真实的关系。**

这就是伪回归的威力——**它让纯粹的噪音看起来像信号。**

## 🧐 为什么非平稳数据会导致伪回归？

理解伪回归的关键，在于理解**OLS 的统计推断为什么会在非平稳数据下崩塌**。

### 平稳性到底是什么？

先明确一个定义。一个时间序列是**平稳的（Stationary）**，如果它的统计性质不随时间变化：

- **均值恒定**：E[Xₜ] = μ（常数）
- **方差恒定**：Var(Xₜ) = σ²（常数）
- **协方差只与时间间隔有关**：Cov(Xₜ, Xₜ₋ₖ) = γₖ（只依赖于 k，不依赖于 t）

对应的，一个序列如果是**非平稳的**，它的均值、方差或协方差会随时间变化。

### 随机游走的"漫游"本质

随机游走 Xₜ = Xₜ₋₁ + εₜ 是最简单的非平稳过程。为什么？

```
Var(Xₜ) = Var(ε₁ + ε₂ + ... + εₜ) = t × σ²
```

随着 t 增大，随机游走的方差**无限增大**。它没有"均值回归"的倾向——一旦漂移到某个位置，它可能就永远回不来了。

### OLS 的崩塌

回想一下 OLS 系数的 t 统计量：

```
t = β̂ / SE(β̂)
```

在经典 OLS 假设下，SE(β̂) 以 1/√n 的速度收敛到 0——样本量越大，估计越精确。

但在非平稳的情况下：
- 两个独立随机游走的**表观相关系数**不会随样本量增加而收敛到 0
- 相反，它在 [-1, 1] 之间随机游走——这就导致了 t 统计量的分布不再是我们熟悉的 t 分布
- t 统计量的尾部变厚——边界值（比如 1.96）不再适用，你需要大得多的阈值才能控制 Type I 错误

> Granger 和 Newbold 发现：在非平稳数据下，**5% 显著性水平的 t 检验实际上有约 75% 的概率拒绝原假设**——也就是说，你得到的显著性大多是假的。

## 🔬 诊断武器：ADF 检验

既然伪回归这么危险，我们怎么知道一个序列是不是非平稳的？

我们需要一个客观的检验——**Augmented Dickey-Fuller（ADF）检验**。

### ADF 检验的核心思想

ADF 检验的原假设是：**序列存在单位根（即非平稳）。**

什么是单位根？回忆一下随机游走的表达式：

```
Xₜ = Xₜ₋₁ + εₜ
```

改写为：

```
Xₜ - Xₜ₋₁ = γXₜ₋₁ + εₜ    （其中 γ = 0）
```

检验 γ = 0 等价于检验序列是否是一个随机游走。如果 γ < 0，序列是平稳的（有均值回归倾向）。

但注意：**γ 的 OLS 估计量在单位根下不再服从 t 分布**——Dickey 和 Fuller 推导了它正确的分布，也就是 ADF 检验使用的临界值。

### Python 实现

```python
from statsmodels.tsa.stattools import adfuller

# ADF 检验一个序列
result = adfuller(X, autolag='AIC')
print(f'ADF 统计量: {result[0]:.4f}')
print(f'p 值:       {result[1]:.4f}')
print(f'临界值:')
for key, value in result[4].items():
    print(f'  {key}: {value:.4f}')
```

对随机游走的输出：

```
ADF 统计量: -1.2345
p 值:       0.6578
临界值:
  1%:  -3.463
  5%:  -2.876
  10%: -2.574
```

**p = 0.6578，不能拒绝原假设。** 无法拒绝"存在单位根"——序列是非平稳的。

对平稳序列的输出（比如白噪声或差分后的序列）：

```
ADF 统计量: -8.7654
p 值:       0.0000
临界值:
  1%:  -3.463
  5%:  -2.876
  10%: -2.574
```

**p ≈ 0.000，拒绝原假设——序列是平稳的。**

> **如何解读 ADF 结果：**
> - p < 0.05 → 拒绝单位根原假设 → 序列 **平稳** ✅
> - p ≥ 0.05 → 不能拒绝单位根原假设 → 序列 **非平稳** ⚠️

### ACF 图：用"自相关衰减速度"来判断平稳性

在正式做 ADF 检验之前，你可以先用 **自相关函数（ACF）图** 快速判断：

- **平稳序列**的 ACF 在滞后几步后迅速衰减到 0（"截尾"）
- **非平稳序列**的 ACF 衰减非常缓慢——即使是滞后 20 步，自相关仍然很大

![ACF 对比](https://cdn.jsdelivr.net/gh/qumin86753649iiu/zhihu-econometric/images/016-adf-demo.png)
*图 2：上排是原始非平稳序列的 ACF——自相关缓慢衰减，即使滞后 20 步仍然显著不为 0。下排是差分后（平稳）序列的 ACF——自相关快速衰减到 0。这种对比是判断平稳性的最直观方法。*

## 🔧 延伸：平稳化——差分法

找到了非平稳的问题，下一步就是修复它。

对于金融时间序列（股价、汇率、利率），最常见的修复方法就是**差分（Differencing）**。

### 一阶差分

```
ΔXₜ = Xₜ - Xₜ₋₁
```

对于随机游走：

```
ΔXₜ = Xₜ - Xₜ₋₁ = εₜ
```

差分后，我们得到了白噪声——完美地平稳！

在 Python 中：

```python
# 一阶差分
X_diff = np.diff(X, n=1)

# 检验差分后的序列是否平稳
result_diff = adfuller(X_diff, autolag='AIC')
print(f'差分后 ADF p 值: {result_diff[1]:.4f}')
```

一个直观的理解：**随机游走的"步长"（即每天的涨跌幅）是平稳的，但"位置"（即价格水平）不是。** 差分就是去掉"位置"，只保留"步长"。

### 什么时候用一阶差分？

| 场景 | 原始序列 | 差分后 |
|------|---------|-------|
| 股票价格 | 非平稳 | 收益率（平稳） |
| GDP | 非平稳（增长趋势） | 增长率（平稳） |
| 利率 | 通常平稳（有均值回归） | 不需要差分 |
| 汇率 | 通常非平稳 | 变化率（平稳） |

**注意：并不是所有非平稳序列只需要一次差分就变平稳。** 有些序列需要二阶差分（比如某些价格指数）。需要差分几次才能平稳，这个次数称为 **$d$（差分阶数）**，是后面 ARIMA 模型中最重要的参数之一。

![差分前后对比](https://cdn.jsdelivr.net/gh/qumin86753649iiu/zhihu-econometric/images/016-differencing.png)
*图 3：上排是原始非平稳序列——随机游走一直在"游荡"，没有固定均值。下排是差分后的序列——在 0 附近随机波动，方差稳定。差分就像给序列"脱了外套"——去掉了趋势和随机游走的惯性。*

## 💼 量化视角：金融序列的平稳化

在量化交易中，**是否理解平稳性是区分业余和专业的标志之一。**

### 一个常见错误

很多刚入门的量化研究者会直接把股票价格作为因变量跑回归——这是经典的新手错误。

```python
# ❌ 错误做法：用价格做回归
sm.OLS(stock1_price, stock2_price).fit()   # 伪回归！

# ✅ 正确做法：用收益率（差分）做回归
sm.OLS(stock1_return, stock2_return).fit()  # 避免伪回归
```

因为**价格是随机游走（非平稳），而收益率是差分（平稳）**。用价格做回归，你很可能发现两只毫不相干的股票"显著相关"。

### 协整：平稳性在配对交易中的应用

故事不止于此。有些时候，**两个非平稳序列的某种线性组合会是平稳的**——这就是**协整（Cointegration）**。

最著名的例子：**可口可乐和百事可乐的股价**。两者各自是非平稳的（随机游走），但它们的价差（或比例）可能是平稳的——因为两家公司的基本面同受饮料市场影响，价差会"均值回归"。

配对交易（Pairs Trading）就是基于协整思想：当价差偏离均值时，做空贵的、做多便宜的，等待价差回归。

当然，协整检验（Engle-Granger 两步法、Johansen 检验）是一个更深的主题——我们会在后续文章中详细展开。

## ✅ 小结

这一篇我们学到：

- **平稳性** = 均值、方差、协方差不随时间变化——这是时间序列分析的基础假设
- **非平稳序列**（如随机游走）会导致**伪回归**——两个毫无关系的序列被误判为显著相关
- **识别方法**：ACF 图（看衰减速度）+ ADF 检验（客观统计检验）
- **修复方法**：差分——把非平稳序列变成平稳序列，对金融序列尤其适用
- **量化启示**：永远不要用价格做回归，用收益率（差分）才可靠
