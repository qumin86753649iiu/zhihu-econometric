# 那条线是怎么画出来的？——最小二乘法的几何直觉

> 上一篇文章我们画了一条穿过散点图的线。这条线是怎么"画"出来的？它不是随手画的。

---

## 🚴 先想一个问题：外卖配送费该收多少？

假设你开了一个配送平台。你发现一个规律：配送距离越远，配送费就越高。

你收集了 20 单的数据：

```
距离 (km) | 配送费 (元)
0.8       | 5.5
1.5       | 7.0
3.2       | 9.5
...       | ...
```

你想找到一个公式：**给定配送距离，合理配送费是多少？**

最直接的想法：在纸上画个散点图，用手比划一下，画一条线穿过这些点。

但问题来了——**每个人手画出来的线都不一样**。你画一条，你同事画一条，谁是对的？

---

## 🎯 "误差"就是你和真相之间的距离

先定义清楚：什么叫"好的线"？

对任何一个点来说，**线预测的配送费"真实配送费 = 误差"**。

- 如果线预测的是 8 元，实际收了 10 元，误差就是 2 元
- 如果线预测的是 12 元，实际收了 10 元，误差就是 -2 元（方向不重要，大小才重要）

**所以一个好的拟合线，就是让所有点上的"误差"总体尽可能小。**

![三条候选拟合线对比](https://cdn.jsdelivr.net/gh/qumin86753649iiu/zhihu-econometric/images/002-candidate-lines.png)
*图 1：三根不同的候选线。左边太平缓（低估远距离），中间太陡峭（高估近距离），右边刚好合适。灰色的竖线就是每个点的"误差"。*

---

## 📐 "最小二乘法"这个名字的来由

数学家也碰到了同样的问题。他们的方案非常优雅：

> **把所有误差平方后加起来，找到让这个"平方和"最小的那根线。**

为什么是"平方"而不是绝对值？

两个原因：

1. **平方放大大的误差** — 一个误差 5 元的点的平方是 25，比两个误差 2.5 的点的平方和（6.25+6.25=12.5）要大得多。平方让模型更在意"犯大错"，而不是容忍几个小错换一个大错
2. **数学上容易求最优解** — 平方和的曲线是一个光滑的碗状，有且仅有一个最低点（导数为零的地方），用微积分就能精确算出来

这就是 "Least Squares"（最小二乘）这个名字的来由——"least" 是最小，"sum of squares" 是误差平方和。

**翻译成大白话：**

> 在所有可能的直线中，找到那一条让"所有点到线的垂直距离的平方和"最小的线。

### 用数学语言写出来：

```
第 i 个数据点：
  真实值 = yᵢ
  线的预测值 = ŷᵢ = β₀ + β₁ × xᵢ
  误差 = yᵢ - ŷᵢ

我们要最小化：
  SSE = Σ(yᵢ - ŷᵢ)²
```

这个 SSE（Sum of Squared Errors）就是"误差平方和"——所有误差平方后的总和。OLS 的全称是 Ordinary Least Squares，翻译过来就是"普通最小二乘法"。

---

## 📊 用 Python 来见证"最小化"的过程

回到我们的配送费数据。先手写一个"最小二乘法"的代码——你不用统计学包，5 行 numpy 就能做：

### 造数据

```python
import numpy as np
import pandas as pd

np.random.seed(2024)
n = 20
distance = np.random.uniform(0.5, 10, n)
# 真实关系：3元底价 + 2元/公里
fee = 3.0 + 2.0 * distance + np.random.normal(0, 2.5, n)
fee = np.maximum(fee, 4.0)  # 配送费不低于4元

df = pd.DataFrame({'distance_km': distance, 'fee_yuan': fee})
```

### 手写最小二乘法（闭式解）

```python
# 最小二乘法的闭式解公式：β = (X'X)⁻¹ X'y
X = np.column_stack([np.ones(n), df['distance_km']])
beta = np.linalg.inv(X.T @ X) @ X.T @ df['fee_yuan']

print(f"截距 (β₀) = {beta[0]:.2f}")
print(f"斜率 (β₁) = {beta[1]:.2f}")
# 输出：截距 = 3.75, 斜率 = 1.76
```

就 3 行核心运算。这不魔法——它做的就是"在所有直线中找出误差平方和最小的那条"。

### 验证一下误差平方和

```python
def compute_sse(b0, b1):
    y_pred = b0 + b1 * df['distance_km']
    return np.sum((df['fee_yuan'] - y_pred) ** 2)

# 三条候选线
print(f"太平缓的线: SSE = {compute_sse(0.5, 3.5):.0f}")
print(f"太陡的线:   SSE = {compute_sse(5.0, 1.5):.0f}")
print(f"OLS 结果:   SSE = {compute_sse(beta[0], beta[1]):.0f} ← 最小！")
```

输出：

```
太平缓的线: SSE = 1134
太陡的线:   SSE = 304
OLS 结果:   SSE = 89 ← 最小！
```

差距是数量级的——错误线和平滑线之间差了十几倍的误差平方和。

## 👁 用眼睛看到"平方"的几何意义

为什么叫"平方"和？看看下面这张图：

![误差可视化：每个点的误差](https://cdn.jsdelivr.net/gh/qumin86753649iiu/zhihu-econometric/images/002-residuals.png)
*图 2：对那条太平缓的线，红色竖线是每个点的误差。每个点旁边标了具体的误差值。左下角显示这条线的 SSE 是 1134。*

再看 OLS 找到的最佳线：

![OLS 找到的最佳拟合线](https://cdn.jsdelivr.net/gh/qumin86753649iiu/zhihu-econometric/images/002-ols-best.png)
*图 3：橙色半透明方块是"误差平方"的几何表示——每个方块的面积 = 误差²。把所有方块的面积加起来，这就是 SSE。OLS 就是让这个总面积最小的那根线。*

真实关系是 `fee = 3.0 + 2.0 × km`，OLS 找到了 `fee = 3.75 + 1.76 × km`——非常接近（因为只有 20 个数据点，且加了噪声）。

它之所以不是完美的 3.0 和 2.0，是因为**数据有噪声**。在现实中，你不知道真实的 β₀ 和 β₁ 是多少——你有的只是数据点和一条尽可能好的线。

## 🤖 延伸：为什么机器学习里到处是"平方误差"？

你可能会在机器学习里反复见到"均方误差"（MSE）这个指标——它和 OLS 的 SSE 是一回事，只是除以了样本量 N。

- 线性回归的损失函数：MSE（均方误差）
- 神经网络的回归任务：MSE
- 梯度下降法优化的目标：MSE

它的优点：
- 处处可导（方便用梯度下降）
- 对大误差更敏感
- 有明确的几何意义

它的缺点：
- 对异常值极度敏感（一个离群点可能把整条线拉歪）

如果数据中有极端值（比如一笔 100 公里的配送费，这显然不是普通的情况），最小二乘法的结果就会被这个点"带跑偏"。这时候会用"绝对误差"（MAE）或者"Huber 损失"来替代。

---

## ✅ 小结

这一篇学到的东西：

- **误差平方和（SSE）** 是所有点到拟合线的垂直距离平方的总和
- **最小二乘法（OLS）** 就是找到让 SSE 最小的那条线
- **斜率和截距** 可以用一个闭式公式直接算出来（不需要猜）
- **平方放大误差** — 让模型更在意大偏差
- **真实关系 vs 估计值** — 因为噪声的存在，我们永远只能"接近"真相
