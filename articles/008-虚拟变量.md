# 分类变量也能放进回归？——虚拟变量的构造和陷阱

> 回归方程的变量不全是数字。朝向、季节、行业、性别——这些分类变量怎么放进回归？

---

## 🏠 一个场景：房子的朝向和房价

回到我们的房价数据。除了面积、位置、房龄这些数值变量，还有一个变量没处理过——**朝向**。

一套房子可以是朝南、朝北、朝东、朝西。这 4 个类别，不是一个数字，怎么放进 `y = β₀ + β₁x + ε` 这个方程里？

最简单的直觉是：给朝向赋值，比如南=1、北=2、东=3、西=4。

**但这是不对的。** 这种赋值隐含了一个荒谬的假设：朝北比朝南贵"1 个单位"，朝西比朝东贵"1 个单位"——类别之间没有"间隔"，你不能说南和北的距离等于东和西的距离。

---

## 🎯 解决方案：虚拟变量

正确的做法是把一个 k 类分类变量拆成 **k-1 个 0/1 变量**。

每个变量回答一个"是或否"的问题：

| 变量 | 问题 | 取值 |
|------|------|:----:|
| `朝向_北` | 这房子朝北吗？ | 1=是, 0=否 |
| `朝向_东` | 这房子朝东吗？ | 1=是, 0=否 |
| `朝向_西` | 这房子朝西吗？ | 1=是, 0=否 |

那朝南的房子呢？它三个变量都是 0。**"朝南"就是基准组（base group）**——其他三个朝向的效应都是"相对于朝南而言"。

```
房价 = β₀ + β₁×面积 + β₂×朝向_北 + β₃×朝向_东 + β₄×朝向_西 + ε
```

- 如果房子朝南 → 三个虚拟变量都是 0 → 房价只看常数项+面积
- 如果房子朝北 → β₂ 生效 → 价格 = β₀ + β₂ + β₁×面积（比朝南贵/便宜 β₂ 万）

**这就是 k-1 规则：一个 k 类分类变量，只能放 k-1 个虚拟变量。少放的那个是基准组。**

---

## 🧪 用 Python 实现

```python
import pandas as pd
import statsmodels.api as sm

# pandas 一行创建虚拟变量，自动 drop 基准组
df_dummies = pd.get_dummies(df['orientation'],
                            prefix='朝向',
                            drop_first=True).astype(int)

# 拼回原始数据
df_reg = pd.concat([df[['price', 'area', 'location', 'age']], df_dummies], axis=1)

# 跑回归
X = sm.add_constant(df_reg.drop('price', axis=1))
model = sm.OLS(df['price'], X).fit()
print(model.summary())
```

输出关键部分：

```
                 coef    std err          t      P>|t|
朝向_北        -7.92     5.040     -1.572      0.118
朝向_南        13.96     4.639      3.009      0.003
朝向_西        -3.82     6.360     -0.601      0.548
```

等一下——这里没有"朝向_东"的系数。**因为它被 drop 了（基准组）。**

- 基准组（朝东）的房价"基准" = β₀（截距）+ 面积和位置的贡献
- 朝向_北：相对于朝东，便宜 7.92 万（但 p=0.118，不显著）
- 朝向_南：相对于朝东，贵 13.96 万（p=0.003，非常显著）
- 朝向_西：相对于朝东，便宜 3.82 万（不显著）

## ⚠️ 虚拟变量陷阱

如果你不小心把 4 个类别的虚拟变量全部放进回归（没有 drop 基准组），会发生什么？

```
房价 = β₀ + β₁×面积 + β₂×朝南 + β₃×朝北 + β₄×朝东 + β₅×朝西 + ε
```

这里有一个问题：**这四个虚拟变量之和恒等于 1。** 任何一套房子一定属于其中一个朝向。这意味着：

> 朝南 + 朝北 + 朝东 + 朝西 = 1（截距列）

**截距列和这四个虚拟变量线性相关**——OLS 无法唯一确定系数。这就是**完全多重共线性**。

结果：Python 会报错，或者系数变成 NaN，或者标准误无限大。

解决方法很简单：**永远只放 k-1 个虚拟变量。**

## 🔗 延伸：行业虚拟变量和量化因子

虚拟变量在量化金融中最常见的应用是**行业虚拟变量**。

在做多因子选股时，不同行业的股票天然有不同的估值水平。如果不控制行业，你的因子分析可能会被"行业效应"污染。

```python
# 比如你有 10 个行业的股票
df = pd.get_dummies(df, columns=['industry'], drop_first=True)

# 跑 Fama-MacBeth 回归
# 行业虚拟变量控制了这个股票属于哪个行业
```

在计量经济学中，这叫做**行业固定效应**——它排除了"因为行业不同而导致的差异"，让你看到**同一行业内**因子的选股能力。

同样，在 A/B 测试中，你想要控制用户所在的地区、设备类型、用户分层——这些都可以用虚拟变量来实现。

## ✅ 小结

这一篇我们学到：

- **虚拟变量** — 把分类变量转换成 0/1 变量放进回归
- **k-1 规则** — k 个类别放 k-1 个虚拟变量，多放一个则完全共线性
- **基准组** — 被 drop 的那个类别是所有虚拟变量系数的"参考点"
- **控制分类变量** — 虚拟变量让你在回归中控制"不可量化"的因素
- **行业虚拟变量** — 量化金融中最常用的虚拟变量应用

**下一篇预告**：交互效应——咖啡×睡眠——两个变量一起作用时，效果可能不是简单相加。
