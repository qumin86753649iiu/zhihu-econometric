# 交互效应——咖啡×睡眠，当 1+1 > 2

> 有些效应不是"要么咖啡有用，要么没用"——而是"咖啡的效果取决于你睡没睡够"。

---

## ☕ 回到咖啡问题

第 1 篇我们用咖啡举例时留了一个问题：咖啡可能只对某些人有效，对另一些人无效。

这里我们把这个想法做一次完整分析。

假设你想知道咖啡对工作效率的影响。你收集了 150 个程序员的数据：咖啡摄入量、每天睡眠时间、工作效率指数。

直觉告诉你：**咖啡的效果可能取决于睡眠**。

- 睡够 8 小时的人，咖啡让他清醒高效
- 只睡 4 小时的人，咖啡只会让他手抖焦虑

换句话说：**X（咖啡）对 Y（效率）的效应，依赖于 Z（睡眠）的水平。**

这就是**交互效应（Interaction Effect）**。

---

## 模型怎么写？

没有交互项的模型：

```
效率 = β₀ + β₁×咖啡 + β₂×睡眠 + ε
```

这个模型说：咖啡的效应永远是 β₁，不管睡眠多少。

有交互项的模型：

```
效率 = β₀ + β₁×咖啡 + β₂×睡眠 + β₃×咖啡×睡眠 + ε
```

这个模型说：咖啡的效应是 **β₁ + β₃×睡眠**——它随睡眠变化。

β₃ 就是交互项系数：
- β₃ > 0 → 睡眠越多，咖啡效果越好（互补）
- β₃ < 0 → 睡眠越多，咖啡效果越差（替代，但一般不会这样）
- β₃ = 0 → 没有交互（加不加都一样）

## 🧪 Python 实现

```python
import statsmodels.api as sm
import pandas as pd

# 创建交互项（中心化睡眠避免共线性）
df['sleep_centered'] = df['sleep'] - 7
df['coffee_x_sleep'] = df['coffee'] * df['sleep_centered']

# 跑带交互项的模型
X = sm.add_constant(df[['coffee', 'sleep', 'coffee_x_sleep']])
m = sm.OLS(df['productivity'], X).fit()
print(m.summary())
```

输出：

```
                 coef    std err          t      P>|t|
coffee          13.87      1.109     12.509      0.000
sleep            6.37      1.708      3.728      0.000
coffee_x_sleep  12.79      0.630     20.310      0.000
```

**交互项 t=20.3，p<0.001——极其显著。** 确实存在交互效应。

解读：
- **咖啡主效应 = 13.9**：睡眠 7 小时的人，每杯咖啡提升 14 单位效率
- **交互效应 = 12.8**：睡眠每多 1 小时，咖啡的效率提升额外增加 12.8 单位

所以：
- 只睡 5 小时的人：每杯咖啡提升 13.9 + 12.8×(5-7) = **13.9 - 25.6 = -11.7**（咖啡反而有害！）
- 睡 7 小时的人：每杯咖啡提升 **13.9**
- 睡 9 小时的人：每杯咖啡提升 13.9 + 12.8×(9-7) = **13.9 + 25.6 = 39.5**

这就是交互效应的实践价值——它告诉你"在什么条件下这个变量有作用"。

## 🔢 不和交互项的陷阱

如果不加交互项，直接跑 `效率 = β₀ + β₁×咖啡 + β₂×睡眠`，得到的 β₁=15.0（对比带交互时的 13.9）。

这个 15.0 是"平均效应"——它把睡得好和睡得不好的人混在一起平均了。

**但"平均"可能不适用于任何一个人。**

如果你是一个只睡 5 小时的人，平均效应说"咖啡提升 15 单位效率"，但实际上咖啡让你更累（-11.7）。

这就是**不恰当的聚合谬误**（Simpson's Paradox 的一种形式）——不加入交互项，你会错过整个故事。

## 🔗 延伸：量化因子中的交互

交互效应在量化金融中同样重要。

比如经典的**规模×价值交互**：

```
股票收益 = α + β₁×规模 + β₂×价值 + β₃×规模×价值 + ε
```

如果 β₃ 显著为正，说明"小盘价值股"的收益率远高于"大盘价值"和"小盘成长"的简单平均——这个交互本身就是一个因子。

很多量化策略的"独家秘籍"就在于发现了别人没发现的交互效应：某个因子在某些条件下特别有效。

## ✅ 小结

这一篇我们学到：

- **交互效应** — X 对 Y 的效应依赖于 Z 的水平
- **交互项 = X × Z** — 在回归中加入乘积项来建模
- **不加交互项的偏差** — 平均效应可能对所有人都不是最优的
- **边际效应图** — 把交互效应直观地画出来
- **交互在量化中** — 因子之间的协同效应往往是最佳策略的来源
