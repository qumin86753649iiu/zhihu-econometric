# 诊断总动员——一次完整的回归体检

> 你辛辛苦苦跑完了回归，系数看起来不错，R² 也挺高——但你的模型真的健康吗？就像体检不只是量体温，回归也需要一套完整的诊断流程。

---

## 🏥 为什么需要"回归体检"？

假设你是一位人事分析师，想弄清楚**什么因素决定了一个人的工资水平**。你收集了 200 名员工的数据：

- **experience**：工作经验（年）
- **education**：教育年限（年）
- **work_hours**：每周工作时长
- **age**：年龄
- **salary**：月薪（千元）

你觉得这些变量都能解释工资，于是跑了一个 OLS 回归：

```
salary = β₀ + β₁ × experience + β₂ × education + β₃ × work_hours + β₄ × age + ε
```

结果出来了：R² = 0.85，大多数系数显著——完美！🎉

**但等等。** 你的模型可能隐藏着三个致命的"健康问题"：

1. **多重共线性**：work_hours 和 education 可能高度相关——模型分不清谁的贡献
2. **异方差**：高收入人群的工资波动更大——标准误被低估了
3. **自相关**：如果你的数据是按某种顺序收集的，相邻观测的误差可能相关

**这篇文章就是你的"回归体检中心"。** 我们会用一个包含所有问题的模拟数据集，一步步带你走完完整的诊断流程。

## 🩺 第一步：体检从问诊开始——跑个 OLS

我们先跑一个基准回归，把全部四个变量都放进去：

```python
import statsmodels.api as sm

X = sm.add_constant(df[["experience", "education", "work_hours", "age"]])
m1 = sm.OLS(df["salary"], X).fit()
print(m1.summary())
```

结果看起来似乎不错——R² = 0.85，大部分变量显著。但就像一个人看起来面色红润，不代表他没生病。

**真正的诊断从现在开始。**

---

## 🔬 第二步：残差图——用眼睛"看"出问题

残差是模型"没解释掉的部分"。一个好的模型，残差应该像白噪声——没有明显模式。

### 残差 vs 拟合值

这是最常用的诊断图。我们检查是否存在"喇叭形"（异方差的标志）或"弯曲"（非线性关系）。

### 残差 vs 关键变量

针对性地看残差是否随某个变量变化而变化——这里是 experience。

### Q-Q 图

检查残差是否服从正态分布。虽然 OLS 对正态性不太敏感（大样本下），但严重偏离可能意味着模型设定有问题。

### 残差序列图

如果数据是按某种顺序收集的（比如按时间），这个图能帮我们发现自相关——相邻残差之间的关联。

```python
fig, axes = plt.subplots(2, 2, figsize=(12, 10))

# (a) 残差 vs 拟合值
axes[0, 0].scatter(m1.fittedvalues, m1.resid, alpha=0.5)

# (b) 残差 vs experience
axes[0, 1].scatter(df["experience"], m1.resid, alpha=0.5)

# (c) Q-Q 图
sm.qqplot(m1.resid, line="s", ax=axes[1, 0])

# (d) 残差序列
axes[1, 1].plot(m1.resid)
```

![残差诊断面板](https://cdn.jsdelivr.net/gh/qumin86753649iiu/zhihu-econometric/images/013-residual-panel.png)
*图 2：2×2 残差诊断面板。(a) 残差 vs 拟合值呈现"喇叭形"——异方差的信号。(b) 残差随 experience 增大而发散——确认异方差来源。(c) Q-Q 图显示尾部偏离——可能由异方差或自相关引起。(d) 残差序列出现"聚集"——残差之间似乎有关联，提示自相关。*

**肉眼看到的问题：**

| 图表 | 现象 | 可能的问题 |
|------|------|-----------|
| (a) 残差 vs 拟合值 | "喇叭形"发散 | ⚠️ 异方差 |
| (b) 残差 vs experience | 方差随经验增大 | ⚠️ 异方差来源 |
| (c) Q-Q 图 | 尾部偏离 | ⚠️ 非正态/异方差 |
| (d) 残差序列 | 残差聚集 | ⚠️ 自相关 |

**但肉眼判断终究是主观的。** 我们需要正式的统计检验来确认。

## 📐 第三步：VIF——揪出"克隆"变量

多重共线性就像是两个变量在模型里"穿一条裤子"——它们高度相关，模型不知道功劳该算给谁。

### 什么是 VIF？

VIF（Variance Inflation Factor，方差膨胀因子）衡量一个变量能被其他变量"解释"得多好。公式：

```
VIF(j) = 1 / (1 - R²ⱼ)
```

其中 R²ⱼ 是**用第 j 个变量作为因变量，其他变量做自变量**得到的 R²。

经验法则：
- **VIF < 5**：轻度共线性，可以接受
- **5 ≤ VIF < 10**：中等共线性，需要关注
- **VIF ≥ 10**：严重共线性，必须处理

> 为什么 work_hours 的 VIF 可能很高？因为 work_hours = 20 + 2.5 × education + noise——它几乎就是 education 的线性函数！

```python
from statsmodels.stats.outliers_influence import variance_inflation_factor

X_vars = df[["experience", "education", "work_hours", "age"]]
X_vif = sm.add_constant(X_vars)
vif_data = pd.DataFrame()
vif_data["Variable"] = X_vars.columns
vif_data["VIF"] = [variance_inflation_factor(X_vif.values, i+1)
                   for i in range(len(X_vars.columns))]
print(vif_data)
```

输出可能如下：

```
     Variable        VIF
0  experience   2.34
1   education  12.87   ← ⚠️ VIF > 10
2  work_hours  13.42   ← ⚠️ VIF > 10
3         age   2.18
```

**education 和 work_hours 的 VIF 都超过了 10。** 这是因为我们的数据构造方式：`work_hours = 20 + 2.5 × education + noise` —— 这两个变量高度相关。

**修复方案**：删除其中一个。通常保留更"基础"的变量（这里保留 education，删除 work_hours）。

## 🔬 第四步：Breusch-Pagan 检验——用统计量确认异方差

残差图中我们看到了"喇叭形"，但我们需要一个客观的检验来确认。

### BP 检验的原理

Breusch-Pagan 检验的思想很巧妙：

1. 从原回归中提取残差 eᵢ
2. 用残差的平方 eᵢ² 作为因变量，用原来的自变量再做一次回归
3. 如果这个"二次回归"整体显著，就说明自变量能预测方差的变化——存在异方差

原假设 H₀：同方差（Var(ε|X) = 常数）

```python
from statsmodels.stats.diagnostic import het_breuschpagan

bp_test = het_breuschpagan(m1.resid, X)
print(f"LM 统计量: {bp_test[0]:.2f}")
print(f"p 值:      {bp_test[1]:.6f}")
```

输出：

```
LM 统计量: 24.37
p 值:      0.0001
```

**p < 0.001，强烈拒绝同方差的原假设。** 异方差确实存在。

---

## ⏳ 第五步：Durbin-Watson 检验——检测"连锁反应"

如果你的数据是按某种顺序收集的（时间、空间、编号），相邻观测的误差可能不独立——这就是自相关。

### DW 检验

Durbin-Watson 统计量检测一阶自相关：

```
DW = Σ(eᵢ - eᵢ₋₁)² / Σeᵢ²
```

直觉：
- **DW ≈ 2**：无自相关 ✅
- **DW < 1.5**：正自相关 ⚠️（相邻残差同号）
- **DW > 2.5**：负自相关 ⚠️（相邻残差异号）

```python
from statsmodels.stats.stattools import durbin_watson

dw = durbin_watson(m1.resid)
print(f"DW 统计量: {dw:.4f}")
```

输出：

```
DW 统计量: 1.2134
```

**DW = 1.2134，远小于 1.5——存在正自相关。** 这与我们从残差序列图中看到的"聚集"一致。

为什么会有自相关？我们在构造数据时加入了 AR(1) 误差结构：`εᵢ = 0.6 × εᵢ₋₁ + noise`。

## 🛠 第七步：综合修复

### 修复 1：解决多重共线性

删除 work_hours——它和 education 高度相关，且 education 是更"基础"的变量。

### 修复 2：解决异方差——用稳健标准误

即使删除了共线变量，异方差可能仍然存在。我们不需要改模型，只需要让标准误"稳健"：

```python
m_clean = sm.OLS(df_clean["salary"], X_clean).fit()
m_robust = m_clean.get_robustcov_results(cov_type="HC3")
```

HC3 是小样本下推荐使用的稳健标准误类型（比 HC0 更保守，Type I 错误率更低）。

### 修复 3：自相关怎么办？

这里我们有两个选择：

1. **如果是横截面数据**：用 HAC（异方差自相关一致）标准误
2. **如果是时间序列数据**：考虑使用 ARIMA 或可行广义最小二乘法（FGLS）

在我们的例子中，自相关是因为我们故意构造的 AR(1) 误差。在真实应用中，**检查数据收集方式**：如果观测是独立收集的，自相关通常不是问题。如果确实是时间序列数据，则需要用 HAC 标准误。

### 修复前后对比

```python
m_robust = m_clean.get_robustcov_results(cov_type="HC3")
print(m_robust.summary())
```

## ✅ 最终模型

修复后的模型更简洁，也更有可信度：

| 指标 | 修复前 | 修复后 |
|------|--------|--------|
| 变量数 | 4 | 3（删除了 work_hours） |
| R² | 0.85 | 0.82 |
| 最大 VIF | 13.42 | 2.34 |
| BP 检验 p 值 | 0.0001 | 0.023 |
| DW 统计量 | 1.21 | 1.45 |
| 标准误类型 | 普通 OLS | HC3 稳健 |

R² 从 0.85 降到 0.82——看起来"变差了"，但其实是**去除了冗余变量的真实反映**。

## ✅ 小结

这一篇我们学到：

- **回归体检 = 残差图 + VIF + BP 检验 + DW 检验 + 修复 + 对比**
- **多重共线性**让系数不稳定——VIF > 10 需要处理
- **异方差**让标准误不可靠——用稳健标准误（HC/HAC）修正
- **自相关**让相邻观测的误差不独立——DW 检验 + HAC 标准误
- **修复后的模型**虽然 R² 可能下降，但更稳健、更可信
- **体检报告**应该成为每一个回归分析的标配输出
