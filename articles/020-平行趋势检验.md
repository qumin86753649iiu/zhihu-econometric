# 平行趋势——DID 的生命线

> 你修了地铁，房价涨了。但你怎么知道涨是因为地铁，而不是因为这两个城市的房价本来就在朝不同的方向走？

---

## 🚇 修地铁前，两条线已经不一样了吗？

第 19 篇我们学了双重差分（DID）——通过比较处理组和对照组在政策前后的变化差异来识别因果效应。

但是，DID 有一个**核心前提**，它不是一个可有可无的假设，而是整个 DID 大厦的地基：

**平行趋势假设（Parallel Trend Assumption）**：在没有政策干预的情况下，处理组和对照组的结局变量应该沿着相同的趋势变化。

什么意思？

想象一下：A 城市修了地铁，B 城市没修。你用 DID 算出地铁让房价涨了 15%。

但是如果 —— 在修地铁之前的三年里，A 城市的房价本来就在以每年 8% 的速度往上涨，而 B 城市每年只涨 3%——那你怎么知道地铁通车后那 15% 的差距，有多少是地铁的功劳，有多少只是两条趋势线的"自然延续"？

**平行趋势假设就是要求：在政策发生之前，处理组和对照组的趋势是平行的。** 这样，我们才能把政策后两条线的"分叉"归因于政策本身。

这就是为什么说：**平行趋势是 DID 的生命线。** 如果这个前提不成立，整个 DID 的因果故事就站不住脚。

这篇我们就来做两件事：
1. **事件研究图（Event Study Plot）**——把每个时期相对于处理时的系数画出来，直观展示平行趋势和政策动态效应
2. **安慰剂检验（Placebo Test）**——随机分配处理时间，看真实估计在"假处理"分布中的位置

---

## 📊 事件研究图：把时间轴上的效应逐个拆开

### 直觉

DID 给我们的是一个"平均处理效应"——政策前后处理组和对照组的平均差异。但现实中的政策效应很少是瞬间全部释放的：

- 有些政策在实施前就有预期效应（企业提前布局）
- 有些政策在实施后效应逐渐加大（学习曲线）
- 有些政策在实施后效应逐渐衰减（边际递减）

**事件研究法**就是把"平均效应"拆开，估计每个时间点相对于基准期的效应。

具体的回归方程是：

```yaml
Yᵢₜ = β₀ + Σ βₖ · (Treatᵢ × Timeₖ) + μᵢ + λₜ + εᵢₜ
```

其中：
- **Treatᵢ** = 1 表示处理组（修地铁的城市）
- **Timeₖ** = 每个时间期的虚拟变量（以政策前一期为基准，剔除 k = -1）
- **μᵢ** = 个体固定效应
- **λₜ** = 时间固定效应

**关键解读：**
- 政策前各期的 βₖ 应接近 0（或统计不显著）——这就是平行趋势的证据
- 政策后各期的 βₖ 为正且逐渐增大——说明政策效果在释放
- 如果政策前某期的 βₖ 显著不为 0——平行趋势可能被违反，DID 结果存疑

### 代码实现

我们用模拟数据来构建一个完整的事件研究图。

```python
import numpy as np
import pandas as pd
import statsmodels.api as sm
import matplotlib.pyplot as plt

# ===== 统一风格 =====
plt.rcParams['font.sans-serif'] = ['SimHei', 'Microsoft YaHei']
plt.rcParams['axes.unicode_minus'] = False
C_MAIN = '#2E86AB'
C_ACCENT = '#F18F01'
C_RED = '#C73E1D'
C_GREEN = '#4C9F70'

# ===== 模拟面板数据 =====
np.random.seed(42)
T = 10          # 10 期（-5 到 4）
N = 100         # 100 个个体（50 处理组 + 50 对照组）
treat_time = 0  # 第 0 期开始处理

# 创建面板
units = list(range(N))
periods = list(range(-5, 5))
data = []
for u in units:
    treat = 1 if u < 50 else 0
    # 个体固定效应
    mu_i = np.random.normal(0, 0.5)
    for t in periods:
        lambda_t = np.random.normal(0, 0.2)
        # 平行趋势：处理组和对照组在政策前趋势相同
        trend = 0.3 * t
        # 政策效应：处理组在政策后的效应，随时间递增
        effect = 0 if t < treat_time else (1.0 + 0.3 * (t - treat_time))
        y = 10 + trend + mu_i + lambda_t + effect * treat + np.random.normal(0, 0.5)
        data.append({'unit': u, 'time': t, 'treat': treat, 'y': y, 'post': 1 if t >= treat_time else 0})

df = pd.DataFrame(data)
```

![事件研究图](https://cdn.jsdelivr.net/gh/qumin86753649iiu/zhihu-econometric/images/020-event-study.png)
*图 1：事件研究图。横轴是相对于政策实施的时间（t=0 为政策实施当期），纵轴是各期系数 βₖ。政策前各期系数在 0 附近波动且置信区间包含 0，支持平行趋势。政策后系数显著为正且逐渐增大，说明政策效应在逐步释放。*

这张图的解读有几个要点：

1. **政策前（t < 0）的系数**：都在 0 附近，95% 置信区间都包含 0。这就是平行趋势的直接证据——处理组和对照组在政策前没有系统性差异。

2. **政策当期（t = 0）的系数**：开始显著为正，政策效果开始显现。

3. **政策后（t > 0）的系数**：逐期增大，说明地铁的房价拉动效应不是一次性的，而是随着时间（地铁网络完善、沿线商业开发）逐步释放。

4. **基准期（t = -1）被设为参照组**，系数被固定为 0——这是事件研究的标准做法。

## 🎲 安慰剂检验：把你的估计扔进"假处理"的分布里

事件研究图检查了"处理前"的趋势是否平行。但还有一个更有力的检验——**安慰剂检验（Placebo Test）**。

### 直觉

安慰剂检验的想法特别简单：

> 如果我把处理时间随机分配给不同时期，再跑 1000 次 DID，得到的 1000 个"假估计"的分布是什么样的？

如果真实估计在这 1000 个假估计中是一个**极端值**（落在分布的尾部），那就说明真实处理效应不太可能是随机噪声造成的。

如果真实估计落在分布的中间——那就尴尬了，说明随机"造"的处理也能产生类似的效应，你的因果故事就站不住脚了。

### 代码实现

```python
# 安慰剂检验：随机分配处理时间 1000 次
np.random.seed(42)
true_estimate = 1.5  # 真实的 DID 估计值
placebo_estimates = []

for _ in range(1000):
    # 随机为每个个体分配一个假的处理时间
    df_placebo = df.copy()
    fake_treat_time = np.random.choice(periods, size=N)
    for u in units:
        mask = df_placebo['unit'] == u
        tt = fake_treat_time[u]
        df_placebo.loc[mask, 'fake_post'] = (df_placebo.loc[mask, 'time'] >= tt).astype(int)
    
    # 跑 DID
    X_placebo = sm.add_constant(df_placebo[['treat', 'fake_post']])
    X_placebo['treat_x_fake'] = X_placebo['treat'] * X_placebo['fake_post']
    m_placebo = sm.OLS(df_placebo['y'], X_placebo).fit()
    placebo_estimates.append(m_placebo.params['treat_x_fake'])
```

![安慰剂分布图](https://cdn.jsdelivr.net/gh/qumin86753649iiu/zhihu-econometric/images/020-placebo-distribution.png)
*图 2：安慰剂检验。直方图展示了 1000 次随机分配处理时间后得到的 DID 估计值分布。红色竖线是真实的 DID 估计值。如果真实值落在分布的极端尾部——说明真实效应不太可能是随机产生的。*

**解读结果：**

- 安慰剂估计值的分布以 0 为中心（因为随机分配处理时间不会产生系统性效应）
- 真实 DID 估计值（红线）远在分布右侧的尾部
- 计算一个"安慰剂 p 值"：有多少比例的安慰剂估计 ≥ 真实估计？

```python
# 计算安慰剂 p 值
p_placebo = np.mean(np.array(placebo_estimates) >= true_estimate)
print(f"安慰剂 p 值 = {p_placebo:.3f}")
```

如果 p < 0.05，我们就可以说：**在 5% 的显著性水平下，真实处理效应不太可能是由随机因素造成的。**

注意：这里的 p 值和常规回归的 p 值含义不同——
- 常规 p 值：在原假设（β=0）下观察到当前结果的概率
- 安慰剂 p 值：在随机分配处理的零模型下，观察到 ≥ 真实效应的概率

后者更直观，也更贴近"反事实"的思维——如果我们把时间线打乱重来，还能得到这个结果吗？

## 📈 正式检验：预处理趋势的统计推断

除了可视化的方式，我们还可以用统计检验来量化平行趋势的假设是否成立。

### 方法一：事件研究系数的联合 F 检验

事件研究中，我们可以检验"所有政策前系数是否联合为零"：

```python
# 检验政策前所有系数联合为 0
# 原假设 H₀: β₋₅ = β₋₄ = β₋₃ = β₋₂ = 0
# 使用 Wald 检验
hypothesis = 'treat_x_time_m5 = 0, treat_x_time_m4 = 0, treat_x_time_m3 = 0, treat_x_time_m2 = 0'
f_test = m_event.f_test(hypothesis)
print(f"平行趋势 F 检验: F = {f_test.statistic:.2f}, p = {f_test.pvalue:.3f}")
```

如果 p > 0.05，我们不能拒绝"平行趋势"的原假设——这就是平行趋势成立的统计证据。

### 方法二：预处理趋势对比（Pre-trend Plot）

我们也可以把处理组和对照组的均值随时间变化的原始线图画出来，直观对比政策前的斜率。

```python
# 按组和时间计算均值
grouped = df.groupby(['time', 'treat'])['y'].mean().reset_index()

fig, ax = plt.subplots(figsize=(10, 6))
for treat_val, label, color in [(0, '对照组（没修地铁）', C_MAIN), 
                                 (1, '处理组（修地铁）', C_ACCENT)]:
    subset = grouped[grouped['treat'] == treat_val]
    ax.plot(subset['time'], subset['y'], 'o-', color=color, 
            label=label, linewidth=2.5, markersize=8)

ax.axvline(x=0, color='gray', linewidth=1.5, linestyle='--', alpha=0.7)
ax.text(-0.1, ax.get_ylim()[1]*0.95, '政策实施', fontsize=10, ha='right', color='gray')
ax.set_xlabel('时间（相对于政策）', fontsize=12)
ax.set_ylabel('结果变量', fontsize=12)
ax.set_title('平行趋势检验：政策前两组趋势对比', fontsize=14, fontweight='bold')
ax.legend(fontsize=10)
ax.grid(alpha=0.2)
```

## 🧪 一个完整的诊断框架

回到我们开头的例子：**检验修地铁前两城市的房价走势是否平行。**

完整的诊断流程应该是这样的：

| 步骤 | 方法 | 判断标准 |
|------|------|----------|
| **Step 1** | 画原始趋势线 | 肉眼观察政策前两组斜率是否相似 |
| **Step 2** | 事件研究图 | 政策前各期系数置信区间包含 0 |
| **Step 3** | 联合 F 检验 | p > 0.05，不能拒绝"平行趋势" |
| **Step 4** | 安慰剂检验 | 真实估计落在安慰剂分布尾部 |

**四个步骤层层递进**，从"看"到"算"到"证伪"，形成完整的证据链——这就是为什么我们说"平行趋势是 DID 的生命线"。

### 常见的平行趋势失败场景

1. **两组趋势天生不同**：处理组本身就在上升趋势中，对照组停滞——需要改用匹配 DID 或合成控制法
2. **预期效应**：企业在政策实施前就调整了行为——如果预期效应显著，就需要调整事件窗口
3. **政策前期的异常波动**：t = -2 期系数显著，但其他前期不显著——可能是偶然，也可能是数据问题
4. **样本太少**：只有 2 个城市、每个城市 2 个时期——平行趋势无法被检验（需要更多数据点）

## 🔗 延伸：DID 的新发展——当你确实没有平行趋势

如果平行趋势不成立，是不是 DID 就不能用了？

不完全是。近年来计量经济学家发展了一系列"放宽平行趋势假设"的方法：

1. **Callaway & Sant'Anna (2021)**：允许处理时间不同，用"从未处理组"作为对照组，计算组别-时间平均处理效应
2. **Sun & Abraham (2021)**：用"从未处理组"或"最后处理组"作为基准，解决"负权重"问题
3. **合成 DID (Arkhangelsky et al., 2021)**：用合成控制法为每个处理单位构造"合成对照组"，使预处理趋势完美平行

这些方法在 `did`、`staggered`、`sdid` 等 Python/R 包中都有实现。

在量化中，这类方法正在被用于：
- **政策冲击对行业收益的影响**（如"双碳"政策对新能源板块）
- **市场微观结构变化**（如印花税调整的市场效应）
- **因子策略的"事件窗口"检验**——某个信号出现后，多空收益在时间轴上的展开

## ✅ 小结

这一篇我们学到了：

- **平行趋势假设**是 DID 的核心前提，没有它就没有因果推断
- **事件研究图**将政策效应拆解到每个时间点，直观展示平行趋势和动态效应
- **安慰剂检验**通过随机分配处理时间，检验真实效应是否可能是随机噪声
- **联合 F 检验**提供了平行趋势的统计量证据
- 如果平行趋势不成立，可以考虑 **Callaway-Sant'Anna**、**Sun-Abraham** 或 **合成 DID** 等新方法
