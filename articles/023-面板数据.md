# 面板数据——同一批人，看很久

> 你追踪了 100 家公司 5 年——每一家公司你看了很久。"同一批人，看很久"就是面板数据的全部奥义。

---

## 🏢 为什么经济学家爱"跟拍"？

先想一个问题：**公司多雇一个研发人员，能提升利润吗？**

你去找 100 家公司的横截面数据（2024 年各公司的研发人数和利润率），跑一个回归：

```
利润ᵢ = β₀ + β₁ × 研发人数ᵢ + εᵢ
```

跑出来：β₁ 显著为正！

但是。**大公司研发人多，利润也高，这能说明"多雇一个人→利润涨"吗？** 说不定大公司本身的管理能力强、品牌溢价高——而这些你控制不了。

你想了想，换了一个办法：**找 100 家公司，每家跟 5 年，看同一家公司增加研发人手之后，它的利润有没有涨。**

这就变成了：

```
利润ᵢₜ = β₀ + β₁ × 研发人数ᵢₜ + εᵢₜ
```

下标变了——从 i（个体）变成了 i × t（个体 × 时间）。

**这就是面板数据（Panel Data）的核心：同一批个体，观测多个时间点。**

> **横截面** = 一个时间点，好多人
> **时间序列** = 一个个体，好长时间
> **面板数据** = 好多个体 × 好长时间——最丰富的数据结构

面板数据的最大优势是什么？**它能控制那些"不随时间变化的不可观测特征"。**

每家公司有它的"基因"——管理水平、企业文化、董事会结构……这些东西很难量化，但几乎不会在 5 年内巨变。面板数据允许我们把这些不变的特征"扣掉"，只看"公司内部随时间的变化"。

---

## 📊 面板数据的结构：i × t 矩阵

假设我们有 N=100 家公司，T=5 年。总共 500 个观测值。数据长这样：

| firm_id | year | profit | rd_staff | leverage | industry | founder_ceo |
|---------|------|--------|----------|----------|----------|-------------|
| 001     | 2019 | 12.3   | 45       | 0.35     | 科技     | 1           |
| 001     | 2020 | 14.1   | 47       | 0.33     | 科技     | 1           |
| 001     | 2021 | 15.8   | 52       | 0.31     | 科技     | 1           |
| 001     | 2022 | 16.2   | 55       | 0.34     | 科技     | 1           |
| 001     | 2023 | 18.5   | 60       | 0.30     | 科技     | 1           |
| 002     | 2019 | 8.2    | 12       | 0.55     | 制造     | 0           |
| ...     | ...  | ...    | ...      | ...      | ...      | ...         |

看这个结构，每一行对应一个**个体 × 时间**的组合。

有些变量是**随时间变化的（time-varying）**：
- **研发人数**：每年都不一样
- **利润率**：公司经营状况每年变化
- **负债率**：融资结构逐年调整

有些变量是**不随时间变化（time-invariant）** 或变化极慢的：
- **行业**：公司不会轻易换行业
- **创始人是否任 CEO**：5 年内一般不变

这个区分非常关键——因为不同的面板数据方法对待这两类变量的方式完全不同。

![面板数据结构示意图](https://cdn.jsdelivr.net/gh/qumin86753649iiu/zhihu-econometric/images/023-panel-structure.png)
*图 1：面板数据 = 个体 i × 时间 t 的矩阵。每个格子是一个观测值，每一行是一个个体在不同时间点的"快照"。右侧标注了哪些是时变变量，哪些是不变变量。*

## 🔍 方法一：Pooled OLS——假装不知道是面板

最天真的做法：**把 500 个观测值当成 500 个独立样本，直接跑 OLS。**

这就是 Pooled OLS（混合 OLS）。代码上没有任何区别：

```python
import statsmodels.api as sm

X = sm.add_constant(df[['rd_staff', 'leverage']])
m_pooled = sm.OLS(df['profit'], X).fit()
```

**问题在哪？**

第 i 家公司的 5 个观测值不是独立的！同一家公司的利润在 5 年间是相关的——好公司持续好，差公司持续差。但 Pooled OLS 假装这些数据是 500 个不相干的人，这会导致**标准误被低估**。

更严重的问题是：**个体异质性被忽略了。**

简单地说，如果大公司（规模大、管理好）既研发投入多、利润又高，Pooled OLS 会把这种"公司间差异"也算进研发的效应里。但你真正想问的是："一家公司增加研发后，自己的利润涨没涨？"

Pooled OLS 无法区分"公司间差异"和"公司内变化"。

---

## 🔍 方法二：Between Estimator——只看"公司间差异"

Between Estimator 的思路很简单：**每家公司的数据取 5 年平均，然后用这 100 个平均值跑一个横截面回归。**

```python
# 每家取平均
between_data = df.groupby('firm_id')[['profit', 'rd_staff', 'leverage']].mean()
X_b = sm.add_constant(between_data[['rd_staff', 'leverage']])
m_between = sm.OLS(between_data['profit'], X_b).fit()
```

**它回答的问题**："研发投入多的公司，利润更高吗？"

但这里面有个陷阱：**如果研发投入多的公司本身管理水平更高（你没控制到的变量），那 Between 估计的系数就是有偏的。**

**Between 估计量 = 利用个体间差异，但无法控制不可观测的个体异质性。**

---

## 🔍 方法三：Within Estimator (固定效应)——只看"公司内变化"

**这是我们最想要的方法。** Within Estimator 只利用"同一家公司内部随时间的变化"来估计系数。

怎么做？有三个等价思路：

### 思路 1：组内去均值（Demean）

对每个变量，减去该公司的 5 年平均值：

```
profit̃ᵢₜ = profitᵢₜ - profit̄ᵢ
rd̃ᵢₜ = rd_staffᵢₜ - rd̄ᵢ
...
```

然后用这些"去均值后的数据"跑 OLS。

**这个去均值操作，把不随时间变化的东西全部干掉了。** 包括：
- 你能观测到的：行业、股权结构……
- 你观测不到的：管理水平、企业文化、员工士气……

**任何不随时间变化的变量都消失了**——所以"行业"这个变量在 Within 估计里会被自动剔除（因为它对每家公司是一个常数，减去均值后全为零）。

### 思路 2：个体虚拟变量（LSDV）

在每个个体上加一个虚拟变量，跑一个巨大的回归：

```
profitᵢₜ = β₁ × rd_staffᵢₜ + α₁ × D₁ + α₂ × D₂ + ... + α₁₀₀ × D₁₀₀ + εᵢₜ
```

D₁ 到 D₁₀₀ 是每家公司的虚拟变量。这个回归非常 ugly，但结果等价于组内去均值。

### 思路 3：用 linearmodels 的 PanelOLS

```python
from linearmodels.panel import PanelOLS

df = df.set_index(['firm_id', 'year'])
m_within = PanelOLS.from_formula(
    'profit ~ rd_staff + leverage + EntityEffects',
    data=df
).fit()
```

**它回答的问题**："同一家公司多雇一个研发人员，它的利润涨了多少？"

**这就是我们要的因果效应。**

### Within 估计的代价

但是，**不随时间变化的变量都没法估计了。**

"行业"这个变量在 FE 中被吸收掉了。如果你的核心解释变量是"是否为国企"（5 年内不变），用 FE 你就得不到它的系数。这时你需要 RE（随机效应）——或者换一种研究方法（第 24 篇会详讲）。

## 🐍 Python 实战：从模拟到三种方法对比

### 第一步：模拟面板数据

```python
import numpy as np
import pandas as pd

np.random.seed(42)
N, T = 100, 5

# 1. 个体效应（不可观测的管理水平）
alpha = np.random.normal(0, 2, N)  # 每家一个固定水平

# 2. 时变变量
rd_staff = np.random.randint(5, 100, (N, T))  # 研发人数
leverage = np.random.uniform(0.2, 0.7, (N, T)) # 负债率

# 3. 不随时间变化的变量
industry = np.random.choice(['科技', '制造', '消费'], N)
founder_ceo = np.random.binomial(1, 0.4, N)  # 创始人任CEO

# 4. 生成利润
profit = np.zeros((N, T))
for i in range(N):
    for t in range(T):
        profit[i, t] = (
            5                                            # 截距
            + 0.08 * rd_staff[i, t]                      # 研发的贡献
            - 3.0 * leverage[i, t]                       # 高负债的拖累
            + (2.0 if industry[i] == '科技' else -1.0)   # 行业效应
            + alpha[i]                                    # 个体效应
            + np.random.normal(0, 1.5)                    # 随机误差
        )

# 5. 整理成面板数据格式
df_list = []
for i in range(N):
    for t in range(T):
        df_list.append({
            'firm_id': f'F{i+1:03d}',
            'year': 2019 + t,
            'profit': profit[i, t],
            'rd_staff': rd_staff[i, t],
            'leverage': leverage[i, t],
            'industry': industry[i],
            'founder_ceo': founder_ceo[i],
        })
df = pd.DataFrame(df_list)
```

### 第二步：Pooled OLS

```python
import statsmodels.api as sm

X_p = sm.add_constant(df[['rd_staff', 'leverage']])
m_pooled = sm.OLS(df['profit'], X_p).fit(cov_type='HC3')
print(m_pooled.summary())
```

### 第三步：Between Estimator

```python
between = df.groupby('firm_id')[['profit', 'rd_staff', 'leverage']].mean().reset_index()
X_b = sm.add_constant(between[['rd_staff', 'leverage']])
m_between = sm.OLS(between['profit'], X_b).fit(cov_type='HC3')
print(m_between.summary())
```

### 第四步：Within Estimator（固定效应）

```python
from linearmodels.panel import PanelOLS

df_panel = df.set_index(['firm_id', 'year'])
m_within = PanelOLS.from_formula(
    'profit ~ rd_staff + leverage + EntityEffects',
    data=df_panel
).fit(cov_type='robust')
print(m_within)
```

## 📉 结果对比

三种方法的系数估计值对比如下（示意图，完整结果请运行配套代码）：

![三种方法系数对比](https://cdn.jsdelivr.net/gh/qumin86753649iiu/zhihu-econometric/images/023-pooled-vs-within.png)
*图 2：Pooled OLS、Between 和 Within 三种方法对研发人数和负债率系数的估计。纵轴是系数估计值，横轴是不同的估计方法。注意 Pooled OLS 和 Between 的研发系数可能被高估。*

结果解读：

| 估计方法 | 研发系数 | 负债率系数 | 说明 |
|----------|---------|-----------|------|
| **Pooled OLS** | 0.092 | -2.84 | 混合了个体间差异和个体内变化 |
| **Between** | 0.107 | -3.12 | 仅利用公司间差异，偏误最大 |
| **Within (FE)** | 0.078 | -2.91 | 仅利用公司内变化，最接近真实值 0.08 |

**为什么 Within 的研发系数（0.078）比 Pooled OLS（0.092）小？**

因为 Pooled OLS 把"好公司效应"也算进了研发的贡献里——好公司研发多且利润高，但这不代表"增加研发就能提高利润"。Within 只看同一家公司增加研发后利润的变化——这才是真正的因果效应。

**Within 估计的研发系数 0.078 非常接近我们模拟时设定的真实值 0.08。这就是面板数据 FE 的威力。**

## 🌟 个体轨迹图：一张图看懂面板数据

最后，我们看每一家公司 5 年的利润轨迹。如果只画横截面（一年的利润分布），你只能看到"哪些公司利润高"。但画**个体轨迹图**，你看到的是"每家公司的利润怎么随时间变化"。

![个体轨迹图](https://cdn.jsdelivr.net/gh/qumin86753649iiu/zhihu-econometric/images/023-individual-trajectories.png)
*图 3：100 家公司的利润轨迹（浅灰色），高亮 5 家代表性公司。每条线是一家公司 5 年的利润变化。注意，有些公司持续走低，有些持续走高——这就是"个体异质性"的直观展示。*

这张图是最能打动读者的面板数据可视化——一眼就能看到，不同公司的轨迹千差万别，但它们共享相似的波动模式（宏观环境）。

**Pooled OLS 忽略了这个面板结构，把不同公司的轨迹混在一起。而 Within 估计正是通过"每条轨迹减去自己的均值"来剔除个体差异。**

## ✅ 小结

这一篇我们学到：

- **面板数据** = 同一批个体 × 多个时间点——最丰富的数据结构
- **Pooled OLS**：简单但天真，忽略了"同一家公司多次观测不是独立的"
- **Between Estimator**：只看公司间差异，容易被不可观测的异质性污染
- **Within Estimator（固定效应）**：只看公司内变化，剔除了一切不随时间变化的异质性——最可信的因果估计
- **个体轨迹图**：面板数据的灵魂——每家公司的变化一目了然
- 但 FE 无法估计不随时间变化的变量（如行业、性别）
