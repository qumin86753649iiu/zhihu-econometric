# R² 和调整 R²——你的模型到底好不好？

> 跑完回归后，你看到一个 R²=0.88。这算好吗？如果加几个变量变成 R²=0.91，是变得更好还是只是在"骗自己"？

---

## 📏 R² 的本质：你的模型解释了"多少"？

R² 的定义很简单：

> R² = 模型解释的变异 / 总变异

翻译成人话：

> 来看房的一个客户，问你房价大概多少。你做了两个预测：
> 
> **方案一**：什么都不问，直接猜平均价。误差很大。
> **方案二**：你问面积，跑个回归，然后按面积预测。误差小了很多。
> 
> R² 告诉你：用面积代替平均值，误差减少了多少比例。

- R²=0  → 用你的模型不比直接用平均值好（模型无效）
- R²=1  → 模型完美预测每个数据点（现实中不可能）
- R²=0.88 → 模型解释了 88% 的房价变异

看起来越接近 1 越好——但问题来了。

---

## ⚠️ R² 有一个"缺陷"：它从不下降

R² 有一个非常关键的性质：**加一个新变量，R² 要么上升，要么不变——它从不下降。**

即使你加的是一个完全无关的随机噪声变量，R² 也会上升一点点。

为什么？因为 OLS 会给你新变量分配一个"刚好"的系数，让 SSE 减少一点点——哪怕这个减少纯粹是偶然。

**但这样得到的 R² 是虚高的。** 你加 20 个噪声变量，R² 可能从 0.88 涨到 0.91——但你的模型变得更差（过拟合了）。

这就是"调整 R²"登场的理由：

```
调整 R² = 1 - (1 - R²) × (n-1) / (n-k-1)
```

其中：
- n = 样本量
- k = 变量个数

调整 R² 对**每个新增变量"惩罚"** 一单位的自由度。如果加进来的变量没有真正贡献，调整 R² 会**下降**——而不是像 R² 那样假装上涨。

---

## 🧪 实验：加无意义变量，看 R² 和调整 R² 的差距

来看一个直观的模拟。

先用 4 个真实的房价变量（面积、位置、房龄、卧室数）建立起模型，然后开始往里面添加**完全无意义的随机噪声**。

结果：

![R² vs 调整 R²](images/007-r2-comparison.png)
*图 1：左图是添加真实变量时，R²（蓝线）和调整 R²（橙线）同步上升。右图是在 4 个真实变量基础上不断添加随机噪声变量——R² 一路高歌猛进（从 0.89 涨到 0.91），但调整 R² 到达峰值后就开始下跌了。*

右图是整篇文章最核心的一张图。它告诉你：

- **R²**：永远不会下降，加什么变量都涨→**不诚实**
- **调整 R²**：变量有用才涨，没用就跌→**诚实**

在这个例子中，调整 R² 在 4-5 个变量时达到峰值。这正是我们知道的"真实变量只有 4 个"。

```python
import numpy as np
import statsmodels.api as sm

# 4 个真实变量 + 不断添加随机噪声
for n_noise in range(0, 20):
    X = df[['area', 'location', 'age', 'bedrooms']].copy()
    for j in range(n_noise):
        X[f'noise_{j}'] = np.random.normal(0, 1, n)  # 完全无关的噪声
    m = sm.OLS(df['price'], sm.add_constant(X)).fit()
    print(f"总变量: {4 + n_noise}, R²: {m.rsquared:.3f}, 调整R²: {m.rsquared_adj:.3f}")
```

输出（部分）：

```
总变量:  4, R²: 0.895, 调整R²: 0.891
总变量:  5, R²: 0.896, 调整R²: 0.890  ← 调整R²开始下降
总变量: 10, R²: 0.902, 调整R²: 0.888
总变量: 24, R²: 0.913, 调整R²: 0.885  ← 加了一堆噪声后反而更低
```

4 个真实变量时，调整 R² 是 0.891。加了 20 个噪声后，调整 R² 反而跌到了 0.885。

**这就是调整 R² 的"诚实"之处。**

---

## 👁 过拟合：预测值 vs 真实值

再来看一个更直观的比较。

左边是用"仅面积"的简单模型做的预测，右边是用"面积+20个噪声变量"的过拟合模型做的预测：

![过拟合 vs 简单](images/007-overfit-vs-simple.png)
*图 2：两幅图的散点都是"模型预测值 vs 真实值"。如果预测完全准确，点会落在灰色对角线上。简单模型点散落范围大一些，但整体趋势正确。过拟合模型看似"更准"（R²=0.91），但这个准是假的——它在拟合噪声。*

过拟合模型的 R²=0.913，看起来比简单模型的 0.869 好得多。但调整 R² 告诉我们真相：0.885 < 0.891——过拟合模型其实更差。

**如果换一批新数据测试，过拟合模型的表现会暴跌。而简单模型反而更稳健。**

---

## 🧮 调整 R² 的公式直觉

公式不用背，理解直觉就行：

```
调整 R² = 1 - (1 - R²) × (n-1) / (n-k-1)
```

- n = 100（样本量），k = 20（变量数）
- 惩罚因子 = 99/79 ≈ 1.25
- 所以调整 R² ≈ 1 - 1.25 × (1 - R²)

如果 R²=0.913，1-R²≈0.087，乘以 1.25≈0.109，调整 R²≈0.891。
如果 R²=0.895，1-R²≈0.105，乘以 1.05≈0.110（k=4时），调整 R²≈0.890。

**你看：R² 从 0.895 涨到 0.913，但调整 R² 从 0.890 降到了 0.885——惩罚生效了。**

---

## 🤖 延伸：机器学习中的"过拟合"就是同一件事

**R² 和调整 R² 的故事，本质上是"过拟合 vs 泛化能力"的雏形。**

在机器学习中：
- **训练集 R²** = 模型在已有数据上的表现（通常很好）
- **测试集 R²** = 模型在新数据上的表现（通常差一些）

调整 R² 是一个粗糙的"样本内惩罚"方法。更前沿的做法是：
- **交叉验证**：真正在样本外测试模型
- **Lasso/Ridge（正则化）**：在损失函数中直接加入对复杂度的惩罚
- **AIC/BIC**：另一种信息准则，后面会讲到

**但底层思想是一样的：不要相信一个"塞了很多变量"的高 R²。**

---

## ✅ 小结

这一篇我们学到：

- **R² 永不下降** — 加变量只升不降，所以只看 R² 会误导
- **调整 R² 惩罚复杂度** — 加无用的变量会降低调整 R²
- **过拟合** — 模型学到噪声而非信号，换新数据就崩盘
- **R²=0.88 vs 调整R²=0.88** — 如果你的变量很少差距不大；变量很多差距可能很大
- **从 R² 到正则化** — 惩罚复杂度的思想贯穿统计学到机器学习
