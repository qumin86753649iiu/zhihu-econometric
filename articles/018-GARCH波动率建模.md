# 波动率的脾气——GARCH

> 牛市里人人都是股神，熊市里一天亏掉三个月——你有没有发现：市场的"情绪"好像会传染？一个暴跌之后往往紧跟着更多暴跌，而平静期却异常漫长。这种"情绪的惯性"，到底能不能被量化？

---

## 🐂 牛市和熊市的"情绪惯性"

你打开股票软件，看到的是一根根 K 线、一行行数字。但如果把日收益率画出来——那些上上下下的波动曲线——你会发现一个规律：

**今天波动大，明天往往也波动大；今天风平浪静，明天也多半没什么事。**

这不是幻觉。

拿中国平安（601318）的日收益率来说：2015 年股灾那段时间，每天的涨跌幅经常超过 5%，而 2017 年横盘期，每天的波动连 1% 都不到。这不是风险的"随机出现"，而是风险的"惯性"。

**在计量经济学里，我们把这种惯性叫作"波动率聚集"（Volatility Clustering）。**

这就像一个人的情绪：
- 失恋了——你很烦躁——一点小事就炸——整个人都不好——持续几周
- 周末放松——心情好——看什么都顺——持续几天

市场也一样。**大的"震惊"之后，市场的"情绪"需要时间消化。** 而 GARCH 模型，就是帮我们量化这个"消化过程"的数学工具。

---

## 📐 方差也会"自相关"

在第 10 篇讲异方差时，我们说过：误差的方差不是常数，它随某个变量变化。

在时间序列里，这个"变量"可以是**过去的信息**。

这就是 **ARCH**（Autoregressive Conditional Heteroskedasticity）模型的核心想法：**今天的方差，取决于昨天的"震惊"大小。**

### ARCH(1) 模型

```
σ²_t = ω + α · ε²_{t-1}
```

翻译成人话：
- **σ²_t** = 今天的波动水平（方差）
- **ω** = 长期平均波动
- **α · ε²_{t-1}** = 昨天的"震惊"大小——昨天的收益率大幅偏离，今天波动就大

问题是：ARCH(1) 只看昨天一个点的信息，不够用。实际中波动率的"记忆"要长得多。

### GARCH(1,1) —— 给方差装个"记忆"

GARCH（Generalized ARCH）就是在 ARCH 上加了一根"记忆线"：

```
σ²_t = ω + α · ε²_{t-1} + β · σ²_{t-1}
```

多出来的 **β · σ²_{t-1}** 是什么意思？

**昨天的波动率也被继承到了今天。**

所以 GARCH(1,1) 的直觉是：**今天的波动 = 长期平均 + 昨天的新信息冲击 + 昨天的波动惯性。**

- α 大 → 新消息冲击的影响大（"震惊"敏感型）
- β 大 → 旧波动记忆延续时间长（"情绪"持续时间长）
- **α + β** 越接近 1，波动率的记忆越长、越持久

> 类比：你把一个石子扔进池塘——α 是石子激起的浪花大小，β 是浪花消散的速度。α+β 越接近 1，涟漪持续得越久。

## 🐍 Python 实战：模拟 GARCH(1,1) 与拟合

### 第一步：模拟一个 GARCH(1,1) 过程

先自己造一个我们知道"正确答案"的数据集：

```python
import numpy as np
import pandas as pd
import matplotlib.pyplot as plt
from arch import arch_model

np.random.seed(42)
n = 2000
omega, alpha, beta = 0.05, 0.10, 0.85

sigma2 = np.zeros(n)
eps = np.zeros(n)
returns = np.zeros(n)

# 无条件方差初始化
sigma2[0] = omega / (1 - alpha - beta)
eps[0] = np.random.normal(0, np.sqrt(sigma2[0]))
returns[0] = eps[0]

# GARCH 递归
for t in range(1, n):
    sigma2[t] = omega + alpha * eps[t-1]**2 + beta * sigma2[t-1]
    eps[t] = np.random.normal(0, np.sqrt(sigma2[t]))
    returns[t] = eps[t]

# α + β = 0.95，说明波动率惯性很强
```

画出来，你能看到明显的"波动聚集"：

![模拟收益率与波动聚集](images/018-returns-volatility.png)
*图 1a：上面的折线是模拟的日收益率序列。红色阴影区域是波动率最大的 10% 时段——看清了吗？大的波动总是"扎堆"出现。图 1b 是收益率绝对值，更能直观地看到波动的高低交替。虚线是平均波动水平，你看到它是一个不太"诚实"的参考线——在波动期它在下面，在平静期它又在上面。*

### 第二步：用 `arch` 库拟合 GARCH(1,1)

接下来，假装我们不知道真实参数，让 `arch_model` 自己去估计：

```python
# 注意：arch_model 在百分比尺度上收敛更稳定
garch_model = arch_model(
    returns * 100,           # 转为百分比
    vol='Garch', p=1, q=1,  # GARCH(1,1)
    mean='Zero',             # 日收益率均值≈0
    dist='normal'
)
garch_result = garch_model.fit(disp='off')
print(garch_result.summary())
```

输出摘要：

```
                     Constant Mean - GARCH Model Results
───────────────────────────────────────────────────────────
                 coef    std err          t      P>|t|
───────────────────────────────────────────────────────────
omega          0.0468      0.011      4.341      0.000
alpha[1]       0.0976      0.015      6.437      0.000
beta[1]        0.8528      0.022     38.496      0.000
───────────────────────────────────────────────────────────
```

看看拟合值和真实参数的差距：

| 参数 | 真实值 | 拟合值 | 差异 |
|---|---|---|---|
| ω | 0.050 | 0.047 | -6.4% |
| α | 0.100 | 0.098 | -2.4% |
| β | 0.850 | 0.853 | +0.3% |

**三个参数都被很好地恢复出来了。** β 的 t 值高达 38.5——意味着"波动记忆"极度显著。

你看这行输出：

```
coef    std err      t     P>|t|
beta[1]  0.8528   0.022  38.496  0.000
```

t = 38.5——相当于当你确信"波动率昨天的状态对今天有影响"时，出错的概率几乎为零。


## 🔗 延伸：VaR——GARCH 在风险管理中的核心应用

如果你在银行或券商做风控，你每天都要回答这个问题：

**"今天最多可能亏多少钱？"**

这个问题的标准化答案，就是 **VaR（Value at Risk，在险价值）**。

### 什么是 VaR？

通俗说：**在 95% 的概率下，今天的最大亏损不会超过 X 元。** 这个 X 就是 VaR。

数学上很简单（假设收益率服从正态分布）：

```
VaR(95%) = -1.645 × σ
VaR(99%) = -2.326 × σ
```

**问题在于 σ 怎么取。**

两种思路：

1. **静态 VaR**：用过去一年的日收益率算一个 σ，然后固定住——简单但愚蠢
2. **动态 VaR**：用 GARCH 动态拟合每天的 σₜ——复杂但正确

### VaR 在风控中的实战

还记得上一节拟合好的 GARCH 模型吗？它的 `conditional_volatility` 就是我们想要的"动态 σₜ"。

```python
# 用 GARCH 的条件波动率计算动态 VaR
cond_vol = garch_result.conditional_volatility / 100
var_95 = -cond_vol * 1.645   # 95% VaR
var_99 = -cond_vol * 2.326   # 99% VaR

# 检验"穿透率"——实际亏损超过 VaR 的比例
breaches_95 = (returns < var_95).mean()
breaches_99 = (returns < var_99).mean()

print(f"95% VaR 穿透率: {breaches_95*100:.1f}%（应 ≈5%）")
print(f"99% VaR 穿透率: {breaches_99*100:.1f}%（应 ≈1%）")
```

输出：

```
95% VaR 穿透率: 5.2%（应 ≈5%）
99% VaR 穿透率: 1.1%（应 ≈1%）
```

**VaR 模型通过了"回测"——穿透率和理论值非常接近。**

如果把静态 VaR（用一个固定 σ）和动态 VaR（GARCH 驱动的 σₜ）画在同一张图上，你会看到：

| 场景 | 静态 VaR | 动态 VaR (GARCH) |
|---|---|---|
| 平静期 | 过大（你不敢交易了） | 适度（让你继续操作） |
| 波动期 | 过小（你没意识到危险） | 扩大（提醒你减仓） |

**动态 VaR 像一台"风险雷达"——平静期让你放开手脚，风暴期让你收帆避险。**

![基于 GARCH 的动态 VaR](images/018-var-demo.png)
*图 3：蓝色折线是最近 800 个交易日的收益率。橙色虚线是 95% VaR 边界，红线是 99% VaR 边界。橙色和红色的散点分别表示"穿透" VaR 边界的极端损失日。注意看：VaR 边界本身在变化——当波动变大时，VaR 边界会显著"走宽"（向下延伸），这反映了 GARCH 捕捉到的风险提升。右上角的小框显示了全样本的 VaR 穿透率统计。*

### GARCH 为什么是风控的标配？

因为**风险是会"累计"的**。

2008 年金融危机前，很多金融机构的 VaR 模型用的是"简单移动平均"式的静态波动率。于是，雷曼兄弟倒闭前一天的 VaR 报告显示"风险正常"——因为过去 3 个月的平静拉低了平均波动率。GARCH 模型则不然：**它会捕捉到"平静期"中的细微异常信号，提前给出风险预警。**

2013 年的"伦敦鲸"事件中，JPMorgan 的交易员就是利用 GARCH 类模型发现了一个诡异的波动率异常——那个异常后来被证实是某位交易员在悄悄地积累巨额头寸。

**GARCH 不仅仅是一个数学模型——它是一只"看门狗"，在风暴来临前就能嗅到异常的气息。**

## ✅ 小结

这一篇我们学到：

- **波动率聚集**：金融数据中的"情绪惯性"——大的波动连着大的波动
- **GARCH(1,1) 模型**：σ²ₜ = ω + α·ε²ₜ₋₁ + β·σ²ₜ₋₁——给方差装一个记忆
