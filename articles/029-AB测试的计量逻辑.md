# A/B 测试背后的计量逻辑

> 你花了两个月做了一个新推荐算法，A/B 测试显示 p = 0.03，兴高采烈地上线了。然后发现指标纹丝不动。
> 你真的理解那个 p = 0.03 是什么意思吗？

---

## 🧪 一个按钮的颜色，凭什么值一个亿？

假设你是一家电商平台的数据分析师。

产品经理跑过来跟你说："我们把'购买'按钮从蓝色改成红色，转化率肯定能涨！"

你说："那就试试呗。"

于是你做了一个**A/B 测试**：

- **对照组（A 组）**：1000 个用户看到蓝色按钮
- **处理组（B 组）**：1000 个用户看到红色按钮

三天后，数据回来了：

```
对照组：1000 人中 50 人购买 → 转化率 5.0%
处理组：1000 人中 58 人购买 → 转化率 5.8%
```

净提升 0.8 个百分点，相对提升 16%。看起来不错。

但你真的能确定这 0.8% 的差异不是**随机波动**造成的吗？

万一只是那 1000 个看到红色按钮的用户恰好心情好呢？再来 1000 个用户，结果会不会反转？

**这就是 A/B 测试最根本的问题：观察到的差异，到底是真实的因果效应，还是纯粹的抽样误差？**

要回答这个问题，你需要**假设检验**——而这正是计量经济学最核心的工具之一。

---

## 🎯 第一步：从 t 检验看 A/B 测试的统计逻辑

在计量经济学中，A/B 测试本质上就是一个**两样本均值差异检验**的问题。

用符号写清楚：

- **Yᵃᶦ**：用户 i 在对照组的转化率（0 或 1）
- **Yᵇᶦ**：用户 i 在处理组的转化率（0 或 1）
- **H₀**：μᵃ = μᵇ（两组没有差异）
- **H₁**：μᵃ ≠ μᵇ（两组有显著差异）

最常用的工具是**独立样本 t 检验**。

t 统计量长这样：

```
t = (x̄_B − x̄_A) / √(s²_A/n_A + s²_B/n_B)
```

其中 x̄ 是均值，s² 是方差，n 是样本量。

回到按钮的例子。让我们用 Python 模拟一下完整过程：

```python
import numpy as np
from scipy import stats

# 模拟 1000 个用户的转化结果（0=未转化，1=转化）
np.random.seed(42)
n = 1000

# 对照组：真实转化率 5%
control = np.random.binomial(1, 0.05, n)
# 处理组：真实转化率 5.8%
treatment = np.random.binomial(1, 0.058, n)

t_stat, p_val = stats.ttest_ind(treatment, control)
print(f"对照组转化率: {control.mean()*100:.1f}%")
print(f"处理组转化率: {treatment.mean()*100:.1f}%")
print(f"t 统计量: {t_stat:.4f}")
print(f"p 值: {p_val:.4f}")
```

输出：

```
对照组转化率: 5.0%
处理组转化率: 5.8%
t 统计量: 2.17
p 值: 0.0302
```

p = 0.03 < 0.05，在 5% 的显著性水平下，你**拒绝原假设**，认为红色按钮确实有效。

这就是教科书上的结论。

但等等——如果 A/B 测试就这么简单，为什么还有那么多上线后翻车的案例？

因为 A/B 测试的"计量逻辑"远不止一个 t 检验那么简单。

## ⚠️ 第二步：多重比较——你还在"偷看数据"吗？

现实中的 A/B 测试远比上面复杂。

产品经理等不了三天就问你了："数据出来了吗？有显著的苗头吗？"你偷看了一眼——"嗯，目前处理组领先 2%，虽然 p = 0.12……"

又过了一天，他再来问。你还看——"p = 0.08 了！快了！"

第 3 天，p = 0.03，你宣布胜利。

**但你刚刚做了一件很危险的事：多次检验。**

每次你偷看数据并做了显著性决策，你就在做一次假设检验。如果 p < 0.05 就算显著，每次检验有 5% 的概率犯第一类错误。做 3 次检验呢？

犯错的概率不是 5%，而是：

```
P(至少犯一次错) = 1 − (1 − 0.05)³ ≈ 14.3%
```

这叫做**多重比较问题（Multiple Testing Problem）**——检验次数越多，发现"假阳性"的概率越大。

### Bonferroni 校正——最简单粗暴的解法

最直接的修复方法是**Bonferroni 校正**：

```
调整后的显著性阈值 α' = α / m
```

其中 m 是检验次数。如果你做了 3 次检验，原本 α = 0.05 的阈值变成：

```
α' = 0.05 / 3 = 0.0167
```

你最后一次偷看时的 p = 0.03 > 0.0167，**不显著了**。

Bonferroni 校正简单、通用，但**过于保守**——当检验次数很多时，它会把真正有效应也杀掉。这是典型的"宁可错杀一千，不可放过一个"。

### FDR 校正——更聪明的选择

一个更现代的替代方案是**控制错误发现率（False Discovery Rate, FDR）**。

FDR 的核心思想是：**我可以接受一定比例的假阳性，只要假阳性的比例可控。**

Benjamin-Hochberg 过程是 FDR 控制最常用的方法。它不调整 p 值的阈值，而是**调整 p 值本身**（或等价地，排序后逐步比较）。

在实践中，如果你在分析中同时检验了 20 个指标（转化率、点击率、停留时长、跳出率……），强烈建议用 FDR 校正而不是 Bonferroni。

```python
from statsmodels.stats.multitest import multipletests

# 假设你检验了 20 个指标，得到原始的 p 值
p_values = [0.03, 0.21, 0.002, 0.15, ...]  # 20 个 p 值

# Bonferroni
bonf = multipletests(p_values, method='bonferroni')
# FDR (BH 过程)
fdr = multipletests(p_values, method='fdr_bh')

print("Bonferroni 显著:", bonf[0])  # 可能只有少数几个显著
print("FDR 显著:", fdr[0])          # 通常更多
```

**实验设计黄金法则：在做 A/B 测试之前，就确定好唯一的主指标（Primary Metric），只看它的 p 值做决策。** 其他指标只作为辅助参考。

## 🌀 第三步：辛普森悖论——当"整体"骗了你

即使你正确做了 t 检验、校正了多重比较，还有一个更隐蔽的陷阱——**辛普森悖论（Simpson's Paradox）**。

我们用一个真实的例子来说明。

假设你做 A/B 测试，看新版推荐算法是否提升了用户点击率。

整体数据：

```
          用户数    点击率
─────────────────────────
对照组    10000     4.5%
处理组    10000     4.2%
```

处理组反而更差。算法被毙了。

但是——你把用户按年龄段分层：

```
年轻用户（占比 20%）：
          用户数    点击率
─────────────────────────
对照组     2000     8.0%
处理组     1000     8.5%    ← 处理组好！

中年用户（占比 50%）：
          用户数    点击率
─────────────────────────
对照组     5000     4.0%
处理组     5000     4.3%    ← 处理组好！

老年用户（占比 30%）：
          用户数    点击率
─────────────────────────
对照组     3000     3.0%
处理组     4000     3.2%    ← 处理组好！
```

**每一层内部处理组都更好！但整体上处理组却更差。**

这就是辛普森悖论。怎么发生的？

因为**分流不均衡**——处理组中有大量点击率本来就低的"老年用户"（4000 人 vs 对照组的 3000 人），拉低了整体均值。

**分层权重变了。** 对照组和处理组在各个年龄段的比例不一样。你把更多"难转化"的用户分到了处理组，所以即使每类用户都有提升，整体也被拉下来了。

### 怎么办？

1. **随机化一定要彻底。** 理想的分流是每个用户独立等概率分到两组，大样本下各层比例应该接近相等
2. **做分层分析。** 不要只看整体，按关键特征（年龄、地区、设备类型）分组看
3. **用回归调整。** 把年龄、地区等特征作为控制变量放入回归，看处理组虚拟变量的系数

```python
import statsmodels.api as sm

# 假设 df 包含: click (是否点击), treatment (是否处理组), age_group (年龄段)
X = sm.add_constant(df[['treatment', 'age_group']])
model = sm.OLS(df['click'], X).fit()
# 处理组系数就是控制了年龄效应之后的净效果
```

这就是计量经济学的思想——**不是简单地做均值比较，而是用回归控制混杂因素。**

![辛普森悖论](images/029-simpson-paradox.png)

## 📐 第四步：效应量——别只看 p 值

p 值告诉你"两组差异是否统计显著"，但没有告诉你"这个差异有多大"。

在大样本下，一个微小的差异也能变得统计显著。比如：

```
样本量 100,000
对照组转化率: 5.00%
处理组转化率: 5.05%
p 值: 0.02（显著！）
```

但 5.05% 比 5.00% 只高了 0.05 个百分点，**业务上毫无意义**。

所以你需要看**效应量（Effect Size）**。

最常用的效应量是 **Cohen's d**：

```
d = (x̄_B − x̄_A) / s_pooled
```

其中 s_pooled 是两组的合并标准差。

Cohen 的建议（经验规则）：
- d = 0.2：**小效应**
- d = 0.5：**中效应**
- d = 0.8：**大效应**

在 A/B 测试中，较小的效应量（d < 0.2）即使统计显著，也大概率不值得投入工程资源去上线。

**好的 A/B 测试报告应该同时报告 p 值和效应量**，而且结论应以效应量为主，p 值为辅。

## ⚡ 第五步：统计功效——你的样本量够了吗？

A/B 测试的另一个常见翻车原因是：**样本量不够，检测不出真实的差异。**

这就引出了**统计功效（Statistical Power）**的概念。

统计功效 = 当真实效应存在时，你能够正确地拒绝原假设的概率。

```
功效 = 1 − β
```

其中 β 是第二类错误（假阴性）的概率。

反过来，功效不够高意味着：**即使你的新算法真的有效，你的实验也很可能发现不了它。**

### 影响功效的三个因素

```
功效 ≈ f(效应量, 样本量, 显著性水平)
```

1. **效应量越大，功效越高**——改进幅度越明显，越容易被检测到
2. **样本量越大，功效越高**——数据越多，估计越精确
3. **显著性水平越宽松（α 越大），功效越高**——但代价是假阳性增加

行业标准是：**功效 ≥ 80%**，即在真实效应存在下，你有 80% 的概率检测到它。

### 实验前的功效分析

在做 A/B 测试之前，你应该先算一下需要多少样本。

```python
from statsmodels.stats.power import TTestIndPower

analysis = TTestIndPower()

# 你想检测的最小效应量是 0.3（Cohen's d）
# 希望功效达到 80%，显著性水平 0.05
n_needed = analysis.solve_power(
    effect_size=0.3, power=0.8, alpha=0.05,
    ratio=1.0, alternative='two-sided'
)

print(f"每组至少需要 {n_needed:.0f} 个样本")
```

输出：

```
每组至少需要 175 个样本
```

如果现实条件不允许你收集到 175 个样本，你的实验很可能就是**统计功效不足**——即使有真实效果也检测不出来。

## 🔗 延伸：A/B 测试与 DID 的联系

仔细想想，A/B 测试和我们在第 19 篇讲的**双重差分（DID）**其实共享同一个灵魂：**构造反事实。**

- **A/B 测试**：用随机分流保证对照组是处理组的完美反事实
- **DID**：用平行趋势假设构造处理组"如果没有被处理"的反事实

当 A/B 测试不可行时（比如无法随机化），DID 是天然的后备方案。

但 A/B 测试有一个 DID 不具备的巨大优势：**随机化。** 随机化自动平衡了所有可观测和不可观测的混杂因素——这是 DID 做不到的。

反过来说：**如果 A/B 测试的随机化被破坏了（比如用户自选择进入处理组），那你就在做"伪 A/B 测试"，结果就像没有平行趋势的 DID 一样不可靠。**

## 🧠 核心框架总结

把 A/B 测试的计量逻辑串起来，就是一套**完整的"先验设计 → 过程监控 → 结果解读"体系**：

```
实验前：
  ├── 确定主指标（唯一！）
  ├── 做功效分析，算最少样本量
  ├── 设计分层随机化方案
  └── 预注册：文档化假设和计划

实验中：
  ├── 按随机化方案分流
  ├── 监控分流均衡性（各层比例）
  └── 不偷看 p 值（或使用序贯检验）

实验后：
  ├── 做 t 检验 / 回归调整
  ├── 检查辛普森悖论（分层分析）
  ├── 报告效应量（Cohen's d）
  ├── 校正多重比较（如需要）
  └── 判断业务显著性（≠ 统计显著性）
```

## ✅ 小结

这一篇我们学到：

| 概念 | 一句话 |
|------|--------|
| **t 检验** | A/B 测试的核心工具，判断两组均值差异是否显著 |
| **多重比较** | 检验次数越多，假阳性概率越高——Bonferroni 或 FDR 校正 |
| **辛普森悖论** | 整体结论与分层结论相反——因为分流不均衡 |
| **效应量** | p 值 ≠ 业务意义，报告 Cohen's d 让结论更诚实 |
| **统计功效** | 实验前算最少样本量，保证能检测到真实的差异 |
| **与 DID 的关系** | 共享"反事实"思想，但 A/B 测试的优势在随机化 |
