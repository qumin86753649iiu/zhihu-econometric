# 双重差分——修地铁到底有没有让房价涨？

> 城市 A 宣布要修地铁，房价应声而涨。但你怎么知道涨的那部分真的是地铁带来的？万一是整个城市本来就涨了呢？想要回答这个问题，你需要一个"反事实"——而双重差分法就是构造反事实最经典的工具。

---

## 🚇 一个现实的问题：地铁真的是"房价助推器"吗？

假设现在是 2018 年。

**城市 A** 正式开通了一条贯穿全市的地铁线。开发商和中介都在喊"地铁一响，黄金万两"。你去查房价数据，发现 A 市的地铁沿线房价在开通后确实涨了不少。

但你是个严谨的人。你问了一句：
>
> **"有没有可能，就算不修地铁，A 市的房价本来就会涨？"**

比如，A 市可能正在经历人口流入、经济快速增长。地铁开通和房价上涨可能只是**同时发生**，而不是**因果关系**。

那怎么办？**找一个对照组。**

**城市 B** 进入了你的视野。B 市和 A 市经济水平相近，人口结构相似，房价走势也很像——但 B 市没有修地铁。

如果地铁真的是原因，那么 A 市的房价涨幅应该**超过** B 市。多出来的那部分，就是地铁的**因果效应**。

这就是**双重差分**（Difference-in-Differences, DID）最朴素的思想——**看 A 市比 B 市多涨了多少。**

---

## 🔍 双重差分的核心直觉——差异的差异

我们用数学语言把这个逻辑固化下来。

把两个城市、两个时期的平均房价记为：

```
                   处理前 (Before)    处理后 (After)
─────────────────────────────────────────────────────
A 市（处理组）        A_before           A_after
B 市（对照组）        B_before           B_after
```

**第一步：算 A 市的变化——"前后差异"**

```
ΔA = A_after − A_before
```

但这个变化里混杂了"地铁的影响"和"城市自身的时间趋势"（比如经济自然增长）。

**第二步：算 B 市的变化——"时间趋势"**

```
ΔB = B_after − B_before
```

B 市没修地铁，它的变化纯粹来自时间趋势。

**第三步：相减——"差异的差异"**

```
DID = ΔA − ΔB = (A_after − A_before) − (B_after − B_before)
```

这个差值就是**地铁对房价的因果效应**。因为我们在扣除时间趋势后，看 A 市是否比 B 市多涨了。

> DID 的思想可以用一个公式说清楚：
>
> **想要因果 → 构造反事实 → 用对照组的"变化"作为处理组的"本应如此" → 多出来的就是因果。**

![DID 事件图](images/019-did-parallel-plot.png)
*图 1：A 市（蓝）和 B 市（橙）的房价走势。前 12 个月两条线几乎平行——平行趋势假设成立。第 13 个月地铁开通后，A 市房价显著跃升，而 B 市继续按原有趋势上涨。两条线之间的差距就是 DID 估计量。*

---

## 📐 DID 的数学形式

上面这个直觉对应一个非常简洁的回归模型：

```
Price = β₀ + β₁ · Treat + β₂ · Post + β₃ · (Treat × Post) + ε
```

其中：
- **Treat** = 1 如果是 A 市（处理组），0 如果是 B 市（对照组）
- **Post** = 1 如果是处理后，0 如果是处理前
- **Treat × Post** = 交互项，DID 的核心变量

各系数的含义：

| 系数 | 含义 |
|------|------|
| β₀ | B 市处理前的基准房价 |
| β₁ | A 市与 B 市在基期的固定差异 |
| β₂ | B 市处理前后的时间趋势 |
| **β₃** | **DID 估计量——地铁的因果效应** |

你可以手动验证：

```
β₃ = (A_after − A_before) − (B_after − B_before)
```

这就是为什么叫做"双重差分"——**第一次差分**移除处理组自身的时间趋势（通过对照组），**第二次差分**移除两组之间的固定差异。

下面这个四柱图把 DID 拆解得清清楚楚：

![DID 估计量分解](images/019-did-estimation.png)
*图 2：四根柱子分别代表两座城市在"处理前/处理后"的均值。橙色箭头 ΔA 是 A 市的变化（含地铁效应 + 时间趋势），红色箭头 ΔB 是 B 市的变化（仅时间趋势）。二者的差值 DID 就是地铁带来的净效应。*

---

## 🚨 最重要的假设：平行趋势

DID 有一个**致命的前提**——如果没有政策干预，处理组和对照组的趋势应该是一样的。

这就是**平行趋势假设**（Parallel Trends Assumption）。

换成大白话：在修地铁之前，A 市和 B 市的房价走势应该是平行的（不要求价格相等，只要求变化趋势一致）。

我们看图 1 的前 12 个月——两条线几乎是平行的。这不是偶然的：**在模拟数据时，我们让两座城市共享同一个时间趋势（trend = 0.5）。**

在真实研究中，平行趋势的检验是一个重要的前置步骤（我们会在下一篇文章中专门讨论）。常用的方法包括：
- 画事件图（Event Study Plot），看处理前的系数是否显著
- 做安慰剂检验（Placebo Test），假装政策在不同时间发生

## 🐍 Python 实战：模拟 DID

光说不练假把式。我们用 Python 模拟一个完整的 DID 分析。

### 第一步：模拟数据

```python
import numpy as np
import pandas as pd
import statsmodels.api as sm
import matplotlib.pyplot as plt

np.random.seed(42)

# 参数设置
n_months = 24
months = np.arange(1, n_months + 1)
treatment_month = 13

base_a, base_b = 100, 95     # A 市和 B 市的基期房价
trend = 0.5                   # 平行的时间趋势
treatment_effect = 8          # 地铁的真实因果效应

# 生成 A 市（处理组）房价
price_a = np.zeros(n_months)
for t in range(n_months):
    price_a[t] = base_a + trend * months[t] + np.random.normal(0, 2)
    if months[t] >= treatment_month:
        price_a[t] += treatment_effect

# 生成 B 市（对照组）房价
price_b = np.zeros(n_months)
for t in range(n_months):
    price_b[t] = base_b + trend * months[t] + np.random.normal(0, 2)

# 合并为面板数据
df = pd.DataFrame({
    'month': np.tile(months, 2),
    'price': np.concatenate([price_a, price_b]),
    'treat': [1] * n_months + [0] * n_months,
    'post': [1 if m >= treatment_month else 0 for m in months] * 2
})
df['treatXpost'] = df['treat'] * df['post']
```

我们生成了一个**伪面板数据**——两座城市各 24 个月的房价记录。A 市在第 13 个月受到"地铁开通"的干预。注意两座城市共享相同的时间趋势（trend = 0.5），这是我们埋下的**平行趋势**。

### 第二步：跑 DID 回归

```python
X = sm.add_constant(df[['treat', 'post', 'treatXpost']])
model = sm.OLS(df['price'], X).fit()
print(model.summary())
```

输出（简化）：

```
────────────────────────────────────────────────────────
变量               系数       标准误       t值      P>|t|
────────────────────────────────────────────────────────
const            93.352      0.854     109.24    0.000
treat             5.012      1.208       4.15    0.000
post              6.238      1.208       5.17    0.000
treatXpost        8.159      1.708       4.78    0.000
────────────────────────────────────────────────────────
```

### 第三步：解读结果

让我们逐项来看：

- **const (β₀) = 93.35** — B 市处理前的基准房价（趋势中心，接近我们设定的 95）
- **treat (β₁) = 5.01** — A 市和 B 市的固定差异（我们设定了 100 − 95 = 5，几乎完美还原）
- **post (β₂) = 6.24** — B 市处理前后的时间趋势差异
- **treatXpost (β₃) = 8.16** — **DID 估计量！** 真实效应是 8，估计值是 8.16，差 0.16

**β₃ 就是我们的答案：修地铁让 A 市房价平均多涨了约 8.16 单位。**

![回归系数图](images/019-did-regression.png)
*图 3：四个系数的点估计和 95% 置信区间。红色的"treat×post (DID 效应)"是我们的核心关注——它显著大于 0，且不包含 0。这就是地铁存在因果效应的统计证据。*

### 第四步：手动验证

我们再做一个简单的算术验证，确保回归结果和我们的直觉一致：

```python
# 分组均值
y_ab = df[df['treat']==1][df['post']==0]['price'].mean()  # A 市处理前
y_aa = df[df['treat']==1][df['post']==1]['price'].mean()  # A 市处理后
y_bb = df[df['treat']==0][df['post']==0]['price'].mean()  # B 市处理前
y_ba = df[df['treat']==0][df['post']==1]['price'].mean()  # B 市处理后

did = (y_aa - y_ab) - (y_ba - y_bb)
print(f"DID = {did:.3f}")
```

你会发现这个手动计算的 DID 和回归交互项系数**完全一致**。

## 💡 DID 为什么如此流行？

在所有的因果推断方法中，DID 可能是**最受欢迎**的一个。原因在于：

1. **直觉极其清晰**——"差异的差异"这个框架几乎不需要数学门槛
2. **数据要求相对宽松**——只需要两期（或更多期）的面板数据
3. **可以用回归实现**——一个交互项就搞定了
4. **可以和事件研究法结合**——画出动态效应，直观展示

它在学术研究和政策评估中随处可见：

- 经济学：最低工资对就业的影响（Card & Krueger 1994 的经典论文）
- 公共政策：医保改革对健康的影响
- 金融：注册制改革对 IPO 定价效率的影响
- 量化：某因子在新市场推出前后的表现变化

---

## ⚠️ 容易被误用的地方

DID 虽好，但用错的人也不少。以下几个坑要特别注意：

- **平行趋势不成立**——如果两组在事前趋势就不一样，DID 会给出有偏的估计
- **双向固定效应不完整**——在更复杂的多期 DID 中，还需要控制个体固定效应和时间固定效应
- **处理组和对照组的划分不随机**——城市 A 之所以修地铁，可能正是因为它预期房价会上涨（反向因果）
- **溢出效应**——B 市居民的房价预期可能也受到 A 市地铁的影响，从而破坏了对照组的"纯洁性"

这些内容会在后续的几篇文章中展开讨论。

## 🔗 延伸：DID 在量化研究中的身影

你可能觉得"地铁修不修"是个政策问题，和你做的数据分析没关系。

但 DID 的逻辑其实无处不在。

举个例子：你想评估"某个选股因子在 2020 年之后是否失效了"。

- 处理组 = 在该因子上暴露高的股票
- 对照组 = 在该因子上暴露低的股票
- 处理时间 = 2020 年 1 月
- DID = (高暴露组 after − high before) − (低暴露组 after − 低暴露组 before)

如果 DID 显著为负，说明该因子的超额收益确实下降了——不是因为整个市场变了，而是因为**该因子相对其他因子**的收益变化了。

同样的逻辑也可以用来看：**新规实施后，某类股票的流动性是否比其他股票下降得更多。**

DID 在金融和商业分析中的应用远比你以为的广泛。

## ✅ 小结

这一篇我们学到：

- **双重差分** = 用"对照组的变化"作为"反事实"，去除时间趋势后看处理组的净效应
- **数学形式** = 一个含交互项 (Treat × Post) 的线性回归，交互项系数就是 DID 估计量
- **核心假设** = 平行趋势——处理前两组的变化趋势一致
- **Python 实现** = 模拟面板数据 → OLS 回归 → 解读交互项
- **应用场景** = 政策评估、事件研究、因子时效性分析
