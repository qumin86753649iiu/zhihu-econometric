# 模型选择——少即是多

> "This model has too many variables" —— 这可能是你听过最多的批评，也是最难反驳的。
> 到底多少个变量才算"太多"？AIC 和 BIC 给了你一个数字答案。

---

## 🏠 真实的困境：你面前摆着 12 个候选变量

想象你在做房价预测研究。你手上有这些数据：

- **明显有关的**：面积、卧室数、楼层、地段评分
- **不确定的**：装修年份、距地铁站距离、物业费、绿化率
- **看起来不太相关的**：小区名字的拼音长度、业主平均星座数……

问题是：**你怎么决定到底用哪些变量？**

如果你全都用上，R² 肯定会高——我们第 7 篇说过，加变量 R² 只会涨不会跌。但你真的需要那个"小区名字拼音长度"吗？

> 这就是模型选择的核心困境：**拟合度 vs 简洁性**。
> 
> 更好的拟合度可能只是"记住"了噪声（过拟合），更简洁的模型虽然牺牲一点 R²，但泛化能力更强。

---

## 🧠 AIC 和 BIC 的核心直觉：给复杂度"定价"

调整 R² 是我们第一次尝试给复杂度惩罚。但它有一个数学上的局限：

**调整 R² 的惩罚幅度是"拍脑袋"确定的**——它只是简单地对每个变量扣除一份"固定罚金"。

AIC 和 BIC 更聪明。它们来自信息论和似然函数，有更严谨的理论基础。

### 公式来了（但别怕）

```
AIC = -2ln(L) + 2k
BIC = -2ln(L) + k·ln(n)
```

其中：
- L = 模型的似然值（越大越好）
- k = 参数个数（变量数 + 1 个截距）
- n = 样本量

翻译成人话：

> **第一部分：-2ln(L)** = 模型拟合度（越小越好，越低的意味着模型越能在数据中找规律）
> 
> **第二部分：惩罚项** = 对复杂度的惩罚
> - AIC 的惩罚是 2k（每加一个变量 +2）
> - BIC 的惩罚是 k·ln(n)（每加一个变量 +ln(n)）
> 
> **选择：AIC 或 BIC 最小的那个模型**

因为 n=200 时 ln(200)≈5.3，所以 **BIC 的惩罚（约 5.3/变量）比 AIC（2/变量）更严厉**。这就是为什么 BIC 倾向于选择更简单的模型。

### 直觉理解

想象你在挑选房子：

- **AIC**：你愿意多花一点钱（2 元）租一个更大的房子，只要它让你住得更舒服
- **BIC**：你非常抠门（5.3 元），除非新房间真的很大，否则拒绝

**所以：AIC 倾向于保留更多变量（可能过拟合），BIC 倾向于剔除更多变量（可能欠拟合）。**

---

## 🔬 回到房价数据：看 AIC/BIC 怎么选

我们模拟生成 200 条房价数据，其中只有 **4 个真实信号变量**（面积、卧室数、楼层、地段），另外加入 **8 个完全无关的随机噪声变量**。

然后我们遍历所有 2¹²-1=4095 种可能的变量组合，计算每个模型的 AIC 和 BIC。

![AIC/BIC 模型比较](images/012-model-comparison.png)
*图 1：(a) AIC 随变量个数先降后升，在 5-6 个变量时达到最低；(b) BIC 同样先降后升，但在 4-5 个变量时就达到最低——BIC 的惩罚更严厉。*

看两个关键发现：

**第一，AIC 和 BIC 都是先降后升的 U 型曲线。**
- 刚开始加变量时，拟合度的改善 > 惩罚的增加 → 下降
- 加够变量后，拟合度的改善 < 惩罚的增加 → 上升

**第二，BIC 的最低点更早到来。**
- AIC 最低点在 5-6 个变量
- BIC 最低点在 4-5 个变量
- 我们的真实信号是 4 个，BIC 的"命中"更精确

> 💡 **思考题**：上面的模拟中，4 个真实信号 + 8 个噪声。AIC 选了 5-6 个变量（即包括了 1-2 个噪声），BIC 选了 4-5 个变量（基本只有真实信号）。如果样本量 n 增大到 2000，BIC 的惩罚变成了 ln(2000)≈7.6，它会选得更严格吗？

## 📊 权衡曲线：R² 的谎言

再来看 R² 和调整 R² 在这个场景下的表现：

![R² vs 复杂度权衡](images/012-tradeoff.png)
*图 2：随着变量增加，R²（蓝色）单调上升永不下降；调整 R²（橙色）在 4-5 个变量后开始下降。红色阴影区域（8-12 个变量）是过拟合高风险区。AIC/BIC 在这个转折点附近给出最低值。*

这张图总结了整个系列的核心理念：

| 指标 | 趋势 | 评价 |
|------|------|------|
| R² | 一直涨 | ❌ 不诚实 |
| 调整 R² | 先涨后跌 | ✅ 较诚实 |
| AIC | 先降后升 | ✅ 更严谨 |
| BIC | 先降后升 | ✅ 最严格 |

**没有免费的午餐——想要更低的 AIC/BIC，就要在拟合度和复杂度之间找到平衡点。**

## 🐍 Python 实战：前向选择与最优模型

### 方法一：穷举搜索（当变量少时）

当候选变量少于 15 个时，你可以暴力遍历所有组合：

```python
from itertools import combinations

all_vars = ['area', 'bedrooms', 'floor', 'location',
            'noise_1', 'noise_2', 'noise_3', 'noise_4',
            'noise_5', 'noise_6', 'noise_7', 'noise_8']

results = []
for n_vars in range(1, len(all_vars) + 1):
    for combo in combinations(range(len(all_vars)), n_vars):
        subset = [all_vars[i] for i in combo]
        X = sm.add_constant(df[subset])
        model = sm.OLS(df['price'], X).fit()
        results.append({
            'n_vars': n_vars,
            'vars': subset,
            'aic': model.aic,
            'bic': model.bic
        })

best_aic = min(results, key=lambda x: x['aic'])
best_bic = min(results, key=lambda x: x['bic'])
print(f"AIC 最佳：{best_aic['n_vars']} 变量, AIC={best_aic['aic']:.1f}")
print(f"BIC 最佳：{best_bic['n_vars']} 变量, BIC={best_bic['bic']:.1f}")
```

输出：

```
AIC 最佳：6 变量, AIC=1892.3
BIC 最佳：4 变量, BIC=1912.7
```

### 方法二：前向选择（当变量多时）

当你有 50+ 个候选变量时，穷举会爆炸（2⁵⁰≈10¹⁵ 种组合）。这时候用**前向选择**：

> **流程**：从空模型开始，每一步添加一个使 AIC/BIC 下降最多的变量，直到添加任何变量都不能改善为止。

```python
remaining = all_vars.copy()
selected = []

for step in range(6):  # 演示前 6 步
    best_aic = np.inf
    best_var = None
    best_model = None
    
    for var in remaining:
        candidate = selected + [var]
        X = sm.add_constant(df[candidate])
        model = sm.OLS(df['price'], X).fit()
        if model.aic < best_aic:
            best_aic = model.aic
            best_var = var
            best_model = model
    
    selected.append(best_var)
    remaining.remove(best_var)
    print(f"步骤 {step+1}: 添加 {best_var:12s} | AIC={best_aic:.1f}")
```

![前向选择步骤](images/012-forward-selection.png)
*图 3：前向选择过程中，AIC 和 BIC 随每一步添加变量而下降。前 4 步添加的都是真实信号变量（area、location、bedrooms、floor），之后尝试添加噪声变量，AIC 和 BIC 反而回升——算法自然停止。*

输出展示：

```
步骤 1: 添加 area            | AIC=1935.2 ✅ 信号
步骤 2: 添加 location        | AIC=1918.7 ✅ 信号
步骤 3: 添加 bedrooms        | AIC=1906.1 ✅ 信号
步骤 4: 添加 floor           | AIC=1895.6 ✅ 信号
步骤 5: 添加 noise_3         | AIC=1894.2 ⚠️ 噪声
步骤 6: 添加 noise_7         | AIC=1895.8 ⚠️ 噪声（AIC 回升！停止）
```

看！前 4 步干净利落地找到了所有真实信号变量。第 5 步虽然 AIC 微降（含噪声），但第 6 步 AIC 回升——**前向选择自然告诉你在哪里停止**。

实际中你可以设定停止规则：**当添加任何变量都不能让 AIC 下降超过 1 时，停止。**

## 🔗 延伸：从 AIC/BIC 到正则化（Lasso/Ridge）

**AIC/BIC 是离散选择**——你决定"用"或"不用"某个变量。这是 0 和 1 的二进制决策。

**正则化（Regularization）是连续选择**——你可以"稍微用一点"但不允许它太放肆。

### Ridge 回归（L2 正则化）

Ridge 在 OLS 的损失函数上加了一个惩罚项：

```
Loss = Σ(yᵢ - ŷᵢ)² + λ·Σβⱼ²
```

- λ=0：就是普通 OLS
- λ 很大：所有系数被迫趋近于 0（但不能到 0）

Ridge 不会把变量"删掉"，它只是让那些不太重要的变量**缩到接近 0**。

### Lasso 回归（L1 正则化）

Lasso 用的是绝对值惩罚：

```
Loss = Σ(yᵢ - ŷᵢ)² + λ·Σ|βⱼ|
```

关键差异：**Lasso 可以把系数精确地压缩到 0**——自动帮你做了变量选择。

```python
from sklearn.linear_model import Lasso, Ridge

# Lasso：自动变量选择
lasso = Lasso(alpha=0.5)  # alpha 就是 λ
lasso.fit(X_train, y_train)
print("Lasso 系数：", lasso.coef_)  # 大部分是 0

# Ridge：系数收缩
ridge = Ridge(alpha=0.5)
ridge.fit(X_train, y_train)
print("Ridge 系数：", ridge.coef_)  # 都接近但不等于 0
```

### 三者的对比

| 方法 | 选择方式 | 优点 | 缺点 |
|------|----------|------|------|
| AIC/BIC + 穷举 | 硬选择（0/1） | 理论最严谨 | 变量多时不行 |
| 前向/后向选择 | 启发式硬选择 | 计算快 | 可能错过全局最优 |
| Lasso | 软选择（连续） | 自动+高效 | 需要调 λ |
| Ridge | 收缩但不选择 | 稳定 | 不剔除变量 |

> 在机器学习中，**交叉验证（Cross-Validation）** 是更通用的模型选择方法。但底层思想是一样的：**用样本外的表现而不是样本内的 R² 来评价模型**。

## ✅ 小结

这一篇我们学到：

| 概念 | 一句话 |
|------|--------|
| **模型选择的困境** | 拟合度 vs 复杂度——没有免费的午餐 |
| **AIC** | -2ln(L)+2k——每加一个变量罚 2 分 |
| **BIC** | -2ln(L)+k·ln(n)——每加一个变量罚 ln(n) 分，更严厉 |
| **前向选择** | 从空开始，每次加一个最好的，直到不能改善 |
| **后向消除** | 从满开始，每次删一个最没用的，直到不能改善 |
| **Lasso/Ridge** | 正则化——在损失函数中直接加入复杂度惩罚 |
| **核心理念** | **少即是多——简单的模型往往更好** |
