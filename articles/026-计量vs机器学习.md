# 计量 vs 机器学习——哪个更准？

> 同样的数据，计量学家看到的是"每个变量对结果的影响有多大"，机器学习工程师看到的是"我能把预测误差压到多小"。
> 它们到底谁更"正确"？

---

## 🏠 同一条数据，两种世界观

假设你要预测一套房子的价格。你手上有这些特征：

- **面积**、**卧室数**、**楼层**、**地段评分**、**房龄**
- 外加 4 个无关的噪声变量

现在，一个**计量经济学家**和一个**机器学习工程师**同时来建模。

---

**计量经济学家**会这么做：

1. 打开 statsmodels，跑一个 OLS
2. 看每个变量的系数 —— "面积每大 1m²，房价涨 3 万"
3. 看 p 值 —— "面积显著！房龄显著！噪声变量不显著！"
4. 检查残差、异方差、共线性……
5. 最后给出结论：**"这些变量是这样影响房价的。"**

**机器学习工程师**会这么做：

1. 打开 sklearn，跑一个 RandomForest（或 XGBoost）
2. 调参：树的数量、最大深度、学习率……
3. 看验证集 RMSE
4. 再调参……
5. 最后给出结论：**"我能把房价预测误差控制在 X 万元以内。"**

---

**两种做法都没有错。** 它们只是对"好模型"的定义不同。

本文用同一个房价数据集，让 OLS（计量代表）、Ridge（计量↔ML 过渡）、Lasso（自动变量选择）、RandomForest（ML 代表）同台竞技。

---

## 🤔 计量在乎"参数的含义"

计量经济学的核心问题是：**X 是怎么影响 Y 的？**

当你在回归报告中写下：

```
房价 = 30 + 3.0 × 面积 + 10 × 卧室数 + 0.4 × 楼层 + 18 × 地段 - 2.5 × 房龄
```

这句话的意思是：

> 控制其他所有变量不变，面积每增加 1m²，房价平均增加 3 万元。

这就是**边际效应**——计量经济学最值钱的东西。

有了它，你可以回答商业问题：

- "多修一个卧室，房价能涨多少？" → 约 10 万元
- "房龄老 10 年，跌多少？" → 约 25 万元
- "这个系数显著吗？" → p < 0.01，非常显著

而且，计量经济学家还会告诉你**置信区间**：

> 面积每增加 1m²，房价的**真实效应**有 95% 的概率落在 [2.7, 3.3] 万元之间。

这就是**统计推断**——不是给一个数字，而是给一个**可信的范围**。

---

## 🤖 ML 在乎"预测的精度"

机器学习工程师的回答完全不同。

他们不在乎面积每多 1m² 涨多少钱。他们只在乎一件事：**你给出一套房子的特征，我能不能猜准它的价格。**

RandomForest 说：我可以。

它内部长什么样？300 棵决策树，每棵随机选一部分特征和样本，各自生长到最深 10 层。然后让 300 棵树一起投票（平均）。

你问它"面积的影响是多少？"

它答不上来。它没有一个叫"面积的系数"的东西。

**但它可以在测试集上把 RMSE 做到比 OLS 低 20-40%。**

这就是**预测精度**——ML 最值钱的东西。

---

## ⚔️ 擂台赛：OLS vs Ridge vs Lasso vs RandomForest

我们模拟生成 1000 条房价数据，数据天然包含：

| 结构类型 | 具体内容 |
|---------|---------|
| 线性主效应 | 面积、卧室数、楼层、地段、房龄 |
| 交互效应 | 面积×地段（好地段的大房子溢价）、卧室×楼层（高层大户型溢价） |
| 非线性效应 | 房龄的对数衰减（新房折旧快、老房折旧慢） |
| 噪声变量 | 4 个完全无关的随机变量 |

用 70% 数据训练，30% 测试。结果如下。

### 预测精度对比

![模型 RMSE 对比](images/026-model-comparison.png)
*图 1：四个模型在测试集上的 RMSE 对比。条形高度 = 预测误差（越低越好）。可以看到：OLS ≈ Ridge ≈ Lasso，而 RandomForest 明显领先。*

结果清晰：

| 模型 | 测试集 RMSE | 相对于 OLS 的改善 |
|------|------------|-----------------|
| OLS | 最高（基准） | — |
| Ridge | 略优于 OLS | 小幅改善 |
| Lasso | 接近 Ridge | 小幅改善 |
| **RandomForest** | **最低** | **显著领先** |

为什么 OLS 输了？因为数据中有**交互效应和非线性结构**。

OLS 假设所有变量的影响是**线性且可加**的——面积 x 地段的溢价效应没有被捕捉到，房龄的衰减曲线也没被拟合。除非你在 OLS 中手动添加交互项 `area:location` 和对数项 `log(age+1)`，否则 OLS 永远学不到这些模式。

**Ridge 和 Lasso 呢？** 它们本质还是线性模型，只是加了正则化（第 12 篇讲过）。正则化让系数更稳定，但无法突破线性假设的局限。

**RandomForest 为什么赢？** 树模型天然能捕捉交互和非线性——不需要你告诉它"面积和地段可能交互"，它自己会学。这就是 ML 最大的优势：**自动发现复杂模式。**

### 逐个看预测效果

![实际 vs 预测散点图](images/026-prediction-scatter.png)
*图 2：四个模型的预测值 vs 实际值散点图。点越接近对角线（y=x），预测越准。OLS/Ridge/Lasso 的散点离散度类似；RandomForest 的点更集中在对角线附近。*

注意看：

- **OLS**：散点沿对角线分布，但在高房价区域（右上角）存在系统性偏差——因为高房价往往来自交互效应，OLS 没学到
- **Ridge/Lasso**：与 OLS 类似，但系数被收缩，高房价区域的偏差略有改善
- **RandomForest**：散点明显更紧贴对角线，高房价区域的偏差也小得多

## 🔬 可解释性的较量

预测精度只是故事的一半。

如果 ML 预测更准，我们是不是就应该全盘抛弃 OLS，只用 RandomForest？

**不。因为你还需要解释模型。**

### OLS：完整的统计推断

![可解释性对比](images/026-interpretability.png)
*图 3：三种可解释性视角的对比。(a) OLS 系数带 95% 置信区间——你能看到每个变量效应的大小、方向和统计显著性；(b) Ridge 系数被 L2 正则化收缩但未归零；(c) RandomForest 特征重要性——只能告诉你"这个特征重要"，不能告诉你"它怎么影响房价"。*

图 3(a) 是 OLS 的独门绝技：**每一个系数的点估计 + 置信区间 + p 值**。

即使你手动在 OLS 中添加了交互项，每个交互项的系数同样有完整的推断。这就是计量的承诺：**每个数字都有统计含义。**

再看 OLS 的 p 值输出：

```python
                系数        p值    显著
area          3.12     0.000    ***
bedrooms      8.45     0.000    ***
floor         0.38     0.014     **
location     17.22     0.000    ***
age          -2.18     0.000    ***
noise_1      -0.03     0.708
noise_2       0.41     0.215
noise_3      -0.27     0.354
noise_4       0.11     0.680
```

看！5 个真实信号变量全部显著（p < 0.05），4 个噪声变量全部不显著（p > 0.1）。**OLS 自动帮你区分了信号和噪声**——这在 ML 中做不到。

### Ridge：系数被"收缩"了

图 3(b) 展示 Ridge 的系数。与 OLS 相比：

- 所有系数的绝对值都变小了（这就是 L2 惩罚的效果）
- 但系数的**相对大小**仍然保持——面积仍然最重要，噪声仍然很弱
- 问题是：**你不能说"面积的系数是 2.1 万/m²，对吧？"**——因为 Ridge 的系数是有偏的，它为了降低预测方差而引入了偏差

### RandomForest：我只能告诉你"重要，但不告诉你怎么重要"

图 3(c) 是特征重要性。它告诉了你三件事：

1. **面积最重要**（接近 50% 的重要性权重）
2. **地段和房龄也重要**
3. **噪声变量被自动忽略**（重要性接近 0）

但它**没有告诉你**：

- 面积的边际效应是多少？（涨 3 万？还是 2 万？还是非线性地变化？）
- 面积和地段的交互到底怎么影响房价？
- 房龄的具体衰减曲线是什么形状？

**这就是"可解释性"的代价：你知道"什么重要"，但不知道"怎么重要"。**

## ⚖️ 权衡三角：精度、可解释性、灵活性

把四个模型放在一张二维坐标上：

```
                   高可解释性
                      │
                      │ OLS
                      │
                      │ Ridge
                      │
                      │ Lasso
                      │
                      │ RandomForest
                      │
                      └───────────────────
                        高预测精度
```

**从左上到右下：可解释性在降低，预测精度在上升。**

这个权衡不是绝对的，但在绝大多数实际问题中，它是一个**普遍规律**：

| 场景 | 推荐模型 | 原因 |
|------|---------|------|
| 写论文、做政策建议 | OLS（或加上交互项/非线性项） | 需要系数解读和显著性判断 |
| 做量化因子选股 | OLS（Fama-MacBeth） | 需要因子暴露的含义 |
| 预测房价、销量 | RandomForest / XGBoost | 预测精度是最重要的 KPI |
| 做风控模型 | Logistic + Ridge | 需要一定可解释性 + 稳定性 |
| 混合方案 | OLS 做推断 + XGBoost 做预测 | 两套系统的结果交叉验证 |

## 💡 那，到底哪个"更准"？

这个问题本身就是陷阱。

> **计量和 ML 对"准"的定义不同。**
>
> - 计量在乎：**参数的估计准不准**（无偏性、一致性、有效性）
> - ML 在乎：**预测的结果准不准**（泛化误差、RMSE）

所以答案取决于你的问题：

- 如果问题是**"面积每涨 1m²，房价涨多少？"** → OLS 更"准"
- 如果问题是**"这套房子到底能卖多少钱？"** → RandomForest 更"准"

**它们回答了不同的问题。**

---

## 🔗 延伸：混合方案——两个模型一起上

一个实用的建议是：**不要只选一个。**

在真实工作中，最好的做法是：

> **用 OLS 做因果推断 + 用 XGBoost 做预测，两套结果互相验证。**

如果 OLS 说"面积很重要"，XGBoost 的特征重要性也说"面积最重要"——那你就更放心了。如果两者矛盾（比如 OLS 说某个变量显著但 ML 说它不重要），那就值得深挖。

两者的交叉验证，比任何一个模型单独都可靠。

---

## ✅ 小结

这一篇我们学到：

| 概念 | 一句话 |
|------|--------|
| **计量的核心** | 在乎"参数的含义"——系数、p 值、置信区间 |
| **ML 的核心** | 在乎"预测的精度"——最小化 RMSE |
| **为什么 OLS 输给 RF** | 数据中有交互和非线性，线性模型学不到 |
| **为什么 RF 不能替代 OLS** | RF 不提供统计推断——你不知道系数，只知道重要性 |
| **权衡三角** | 可解释性 ↔ 预测精度，不可兼得 |
| **最佳实践** | 两个都用，交叉验证 |
